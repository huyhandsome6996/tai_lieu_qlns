📚 Chào mừng đến trang học đồ án Quản lý Nhà sách

Trang web này được thiết kế để giúp bạn hiểu tường tận mọi thứ trong đồ án quanlynhasach (phiên bản Tkinter). Mỗi khái niệm, mỗi hàm, mỗi cú pháp đều được giải thích chi tiết — vì bạn từng nói mình mất gốc và học không hiểu.

💡 Cách học hiệu quả

Học theo thứ tự từ trên xuống dưới trong sidebar trái. Mỗi bài đều có nút "Bài tiếp theo →" ở cuối để chuyển bài.

Nếu gặp đoạn code nào khó hiểu, hãy copy code sang editor Python (như Thonny, VSCode) rồi chạy thử — đọc code mà không chạy thì khó hiểu lắm.

🎯 Đồ án này là gì?

Đồ án Quản lý Nhà sách là bài kiểm tra cuối kì môn Lập trình Nâng cao. Ứng dụng gồm:

  • Giao diện đồ họa bằng Tkinter (Python thuần, không cần cài thêm).
  • Cơ sở dữ liệu SQLite để lưu mặt hàng + tài khoản người dùng.
  • 9 chức năng từ đăng nhập → thêm/sửa/xóa → tìm kiếm → thống kê → giỏ hàng.
  • Cấu trúc dữ liệu tự cài: DSLK đôi, Stack LIFO, Queue FIFO (không dùng list Python).
  • Thuật toán: Merge Sort O(n log n) — sắp xếp trên con trỏ DSLK.
  • OOP: kế thừa + đa hình với 5 lớp con (Sách, Tạp chí, Báo giấy, Luận văn, Bản thảo).

📋 Lộ trình học (20 bài)

Tổng cộng có 20 bài học, chia làm 4 nhóm. Bạn nên học theo thứ tự:

🧱 Nhóm 1: Kiến thức nền (4 bài)
Python cơ bản → OOP → Tkinter → SQLite. Học xong nhóm này mới hiểu code đồ án.
⚙️ Nhóm 2: Thuật toán (4 bài)
DSLK đôi → Merge Sort → Stack LIFO → Queue FIFO. Đây là phần "tự cài" khó nhất đồ án.
🎛️ Nhóm 3: 9 Chức năng (9 bài)
CN01 Đăng nhập → CN02 Đọc DS → ... → CN09 Giỏ hàng. Mỗi chức năng = 1 folder có 3 file _tt/_ui/_dk.
🎯 Nhóm 4: Tổng thể (2 bài)
Cách main.py khởi động + QuanLyTrungTam điều phối. Bảng phân chia công việc cho 3 thành viên.

📖 Quy ước hậu tố tên file

Trong mỗi folder chức năng (cn01 → cn09), có 3 file:

  • _tt = Thuật toán — logic thuần, không import tkinter.
  • _ui = Giao diện — chỉ vẽ widget Tkinter.
  • _dk = Điều khiển — kết nối _ui với _tt, gắn sự kiện.

🏃 Bắt đầu học ngay

Nhấn vào bài đầu tiên để bắt đầu:

🐍 Bài 1: Python cơ bản
Biến, kiểu dữ liệu, if/else, vòng lặp, hàm, list/dict. Nền tảng trước khi học OOP.
🖼️ Bài 3: Tkinter cơ bản
Cửa sổ, Label, Entry, Button, Treeview, sự kiện. Cú pháp tạo giao diện.
🔗 Bài 5: DSLK đôi
Con trỏ, Node, head, tail, prev, next. Cấu trúc dữ liệu tự cài.

❓ Câu hỏi thường gặp

Tôi rất ngu lập trình, học được không?

Được. Trang này được viết cho người mất gốc. Mỗi cú pháp mới đều được giải thích "cái này là gì, tại sao dùng, viết sao". Bạn đọc chậm + chạy code thử là hiểu.

Tôi cần cài gì để chạy code?

Chỉ cần Python 3.10+ (có sẵn Tkinter). Tải tại python.org. SQLite cũng có sẵn trong Python (module sqlite3).

Tôi có thể chạy thử code ở đâu?

Có 3 cách:

  • Editor: VSCode, Thonny, PyCharm — paste code vào rồi F5.
  • Online: replit.com (chạy Tkinter cần replit GUI).
  • CMD/Terminal: lưu code thành test.py rồi chạy python test.py.
