🧩 Bài 20: Luồng ghép các "cn" thành app thống nhất

Bạn đã học 9 chức năng riêng lẻ (CN01 → CN09). Nhưng làm thế nào 9 cái đó ghép lại thành 1 app chạy được? Bài này sẽ đi từng bước: từ main.py gọi khung_chinh, khung_chinh gọi từng cn → tất cả cùng chia sẻ QuanLyTrungTam.

📖 Mục tiêu bài học

Sau bài này bạn sẽ hiểu rõ:

  • Tại sao 9 cn riêng lẻ lại chạy được cùng nhau trong 1 cửa sổ.
  • File nào là "cái keo" dán 9 cn lại với nhau.
  • Câu lệnh import nào nối chúng lại.
  • Luồng chạy từ lúc user bấm nút "Thêm sản phẩm" đến khi bảng tự refresh.

1. Vì sao cần ghép? — 9 cn là 9 hòn đá, cần 1 chuỗi xâu lại

Mỗi folder cn01_…, cn02_…, …, cn09_… đều tự chạy độc lập được — mỗi cái có UI riêng + Thuật toán riêng + Điều khiển riêng. Nhưng khi user mở app, họ KHÔNG muốn thấy 9 cửa sổ lẻ tẻ. Họ muốn 1 cửa sổ duy nhất, có thanh công cụ ở trên, bảng dữ liệu ở giữa, giỏ hàng ở góc phải, thống kê ở dưới — và 9 nút bấm để dùng 9 chức năng.

Ví dụ:

  • Nút ➕ Thêm trên thanh công cụ → phải gọi cn03.
  • Nút ✏ Sửa → phải gọi cn04.
  • Nút 🗑 Xóa → phải gọi cn05.
  • Nút ↶ Undo → phải gọi cn08.
  • Bảng chính hiển thị dữ liệu → do cn02 vẽ.
  • Khung giỏ hàng bên phải → do cn09 vẽ.
  • Khung thống kê bên dưới → do cn07 vẽ.

Vậy ai là người "biết" tất cả 9 cn và gắn chúng vào đúng vị trí? Đó là 3 file trong folder khung_chinh/.

💡 Đọc bản đồ thế nào

Theo chiều dọc: main.py ở trên cùng (điểm vào) → khung_chinh ở giữa (cái khung) → chuc_nang/cnXX ở dưới (9 chức năng).

Theo chiều ngang: khung_chinh_tt (trái) giữ STATE (dữ liệu), cua_so_chinh_dk (phải) giữ ĐIỀU KHIỂN (luồng). 2 cái chia việc: thằng giữ data, thằng điều phối luồng.

Mũi tên ◂: cua_so_chinh_dk nhận tham chiếu tới QuanLyTrungTam để truy cập state — KHÔNG tạo bản sao, mà dùng chung object.

3. main.py — điểm vào duy nhất

File main.py chỉ dài ~90 dòng, nhưng đó là điểm vào duy nhất của app. Khi bạn gõ python main.py, đây là file đầu tiên chạy.

import tkinter as tk

# Import state container (bộ não app)
from quan_ly_trung_tam import QuanLyTrungTam
# Import UI + Thuật toán đăng nhập
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_ui import CuaSoDangNhapUI
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_tt import BoDangNhap
# Import Điều khiển cửa sổ chính (lớp "keo")
from khung_chinh.cua_so_chinh_dk import DieuKhienCuaSoChinh


def main():
    # 1. Tạo bộ não app + khởi tạo DB, DSLK, Stack, Queue
    qly = QuanLyTrungTam()
    qly.khoi_tao()

    # 2. Tạo cửa sổ root (Tkinter yêu cầu root trước)
    root = tk.Tk()
    root.title("Quản lý nhà sách")
    root.geometry("100x100")  # Tạm nhỏ, sẽ phóng to sau khi đăng nhập

    # 3. Mở form đăng nhập (cn01)
    bo_dn = BoDangNhap()
    form = CuaSoDangNhapUI(root)
    # ... gắn sự kiện Enter + nút Đăng nhập ...
    root.wait_window(form.top)  # Chờ user đăng nhập xong

    # 4. Nếu đăng nhập OK → mở cửa sổ chính
    if form.nguoi_dung:
        qly.nguoi_dung_hien_tai = form.nguoi_dung
        root.deiconify()  # Hiện root
        # TẠO ĐIỀU KHIỂN CỬA SỔ CHÍNH — nó tự vẽ UI + gắn 9 cn
        app = DieuKhienCuaSoChinh(root, qly, form.nguoi_dung)
        root.mainloop()  # Vòng lặp chính — chạy đến khi user đóng
    else:
        root.destroy()


if __name__ == '__main__':
    main()
📌 4 dòng quan trọng nhất

4 dòng dưới đây là toàn bộ sự kết nối giữa main.py và các module khác:

  1. from quan_ly_trung_tam import QuanLyTrungTam — kéo bộ não vào.
  2. from chuc_nang.cn01_..._ui import CuaSoDangNhapUI — kéo UI đăng nhập.
  3. from chuc_nang.cn01_..._tt import BoDangNhap — kéo thuật toán đăng nhập.
  4. from khung_chinh.cua_so_chinh_dk import DieuKhienCuaSoChinh — kéo lớp "keo".

4 dòng import này là tất cả những gì main.py cần biết về 9 cn. main.py KHÔNG cần biết tới cn02, cn03, ..., cn09 — vì DieuKhienCuaSoChinh sẽ lo việc đó.

4. QuanLyTrungTam — bộ não chung

QuanLyTrungTam là singleton (chỉ có 1 object duy nhất trong toàn app). Nó giữ:

  • danh_sach — DSLK đôi chứa toàn bộ sản phẩm.
  • bo_undoredo — dict 2 Stack (undo + redo).
  • gio_hang — Queue FIFO chứa giỏ.
  • nguoi_dung_hien_tai — người đang đăng nhập.

Vì QuanLyTrungTam giữ state toàn cục, mọi chức năng đều cần tham chiếu tới nó. Đó là lý do nó được truyền vào mọi hàm __init__ của lớp Điều khiển:

class DieuKhienCuaSoChinh:
    def __init__(self, root, quan_ly, nguoi_dung):
        self.root = root
        self.quan_ly = quan_ly   # ← nhận tham chiếu QuanLyTrungTam

        # Tạo 9 điều khiển con — mỗi cái đều nhận 1 phần state từ quan_ly
        self.dk_danh_sach = DieuKhienDocDanhSach(
            self.ui.ui_bang, quan_ly.danh_sach)         # ← truyền DSLK
        self.dk_them = DieuKhienThemSanPham(
            quan_ly.danh_sach, quan_ly.bo_undoredo)     # ← truyền DSLK + 2 Stack
        self.dk_sua = DieuKhienSuaSanPham(
            quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_xoa = DieuKhienXoaSanPham(
            quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_tim_sx = DieuKhienTimKiemSapXep(
            self.ui.ui_tim_sx, quan_ly.danh_sach, self.ui.ui_bang)
        self.dk_thong_ke = DieuKhienThongKe(
            self.ui.ui_thong_ke, quan_ly.danh_sach)
        self.dk_undoredo = DieuKhienHoanTacLamLai(
            self.ui.ui_undoredo, quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_gio_hang = DieuKhienGioHang(
            self.ui.ui_gio_hang, quan_ly.danh_sach, quan_ly.gio_hang)  # ← Queue

Đây chính là câu trả lời cho câu hỏi "các cn liên kết qua file nào?" — chúng liên kết qua tham chiếu object được truyền vào __init__ của lớp Điều khiển.

5. khung_chinh — cái khung gầm

Folder khung_chinh/ có 3 file (giống hệt pattern của mỗi cn):

File Lớp Vai trò
khung_chinh_tt.py QuanLyTrungTam Giữ state toàn cục (DSLK, Stack, Queue, user)
cua_so_chinh_ui.py CuaSoChinhUI Vẽ cửa sổ chính: toolbar + bảng + giỏ + thống kê
cua_so_chinh_dk.py DieuKhienCuaSoChinh Lớp "keo" — tạo 9 dk con + gắn command cho 9 nút

5.1. CuaSoChinhUI — vẽ cửa sổ chính

CuaSoChinhUI chỉ vẽ widget, không có logic. Quan trọng: nó import UI của 5 cn để tái sử dụng — không viết lại từ đầu:

import tkinter as tk
from tkinter import ttk

# Import UI của 5 cn con — TÁI SỬ DỤNG
from chuc_nang.cn02_doc_danh_sach.doc_danh_sach_ui import BangSanPhamUI
from chuc_nang.cn06_tim_kiem_sap_xep.tim_kiem_sap_xep_ui import ThanhTimKiemSapXepUI
from chuc_nang.cn07_thong_ke.thong_ke_ui import KhungThongKeUI
from chuc_nang.cn08_hoan_tac_lam_lai.hoan_tac_lam_lai_ui import ThanhUndoRedoUI
from chuc_nang.cn09_gio_hang.gio_hang_ui import KhungGioHangUI


class CuaSoChinhUI:
    def __init__(self, root, ho_ten, vai_tro):
        self.root = root
        root.title(f'Quản lý nhà sách — {ho_ten} ({vai_tro})')
        root.geometry('1100x700+100+50')
        self.vai_tro = vai_tro
        self._tao_giao_dien()

    def _tao_giao_dien(self):
        # === THANH ĐẦU ===
        frame_dau = ttk.Frame(self.root, padding=10)
        frame_dau.pack(fill='x')
        ttk.Label(frame_dau, text='QUẢN LÝ NHÀ SÁCH',
                  font=('Arial', 16, 'bold')).pack(side='left')
        # Khung user bên phải
        frame_user = ttk.Frame(frame_dau)
        frame_user.pack(side='right')
        self.lbl_ten_user = ttk.Label(frame_user, text='')
        self.lbl_ten_user.pack(side='left', padx=5)
        self.btn_dang_xuat = ttk.Button(frame_user, text='Đăng xuất')
        self.btn_dang_xuat.pack(side='left', padx=5)

        # === THANH CÔNG CỤ ===
        frame_cc = ttk.Frame(self.root, padding=(10, 0, 10, 5))
        frame_cc.pack(fill='x')
        self.btn_them = ttk.Button(frame_cc, text='➕ Thêm', width=10)
        self.btn_them.pack(side='left', padx=2)
        self.btn_sua = ttk.Button(frame_cc, text='✏ Sửa', width=10)
        self.btn_sua.pack(side='left', padx=2)
        self.btn_xoa = ttk.Button(frame_cc, text='🗑 Xóa', width=10)
        self.btn_xoa.pack(side='left', padx=2)

        # Thanh Undo/Redo — DÙNG UI CỦA cn08
        self.ui_undoredo = ThanhUndoRedoUI(frame_cc)
        # Thanh Tìm/Sắp — DÙNG UI CỦA cn06
        self.ui_tim_sx = ThanhTimKiemSapXepUI(frame_cc)

        # === KHUNG CHÍNH: BẢNG TRÁI + KHUNG PHẢI ===
        paned = ttk.PanedWindow(self.root, orient='horizontal')
        paned.pack(fill='both', expand=True, padx=10, pady=5)
        frame_bang = ttk.LabelFrame(paned, text='Danh sách sản phẩm', padding=5)
        paned.add(frame_bang, weight=3)
        # BẢNG — DÙNG UI CỦA cn02
        self.ui_bang = BangSanPhamUI(frame_bang)

        # Khung phải: Thống kê + Giỏ hàng
        frame_phai = ttk.Frame(paned)
        paned.add(frame_phai, weight=2)
        self.ui_thong_ke = KhungThongKeUI(frame_phai)   # cn07
        self.ui_gio_hang = KhungGioHangUI(frame_phai)   # cn09
⚠️ Chú ý: 9 cn KHÔNG đều được nhúng trực tiếp vào UI

Chỉ có 5 cn có UI nhúng trực tiếp vào cửa sổ chính: cn02 (bảng), cn06 (thanh tìm), cn07 (thống kê), cn08 (undo/redo), cn09 (giỏ hàng).

4 cn còn lại — cn03 (Thêm), cn04 (Sửa), cn05 (Xóa), cn01 (Đăng nhập) — mở form modal riêng khi user bấm nút. Vì vậy UI của 4 cn này không nằm trong cửa sổ chính.

5.2. DieuKhienCuaSoChinh — lớp "keo" dán 9 cn lại

Đây là file QUAN TRỌNG NHẤT nếu bạn muốn hiểu "làm sao 9 cn lại gộp được". Nó làm 3 việc:

  1. Tạo 9 đối tượng Điều khiển con — mỗi cái là 1 cn.
  2. Gắn command cho 9 nút trên UI → gọi hàm của dk con tương ứng.
  3. Cung cấp refresh_all() — gọi sau mỗi thao tác để cập nhật UI.
# Import UI cửa sổ chính (chính nó)
from .cua_so_chinh_ui import CuaSoChinhUI
# Import UI đăng nhập (để mở lại khi đăng xuất)
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_ui import CuaSoDangNhapUI
# Import 7 Điều khiển chức năng con (cn01 không cần vì đăng nhập xử lý ở main.py)
from chuc_nang.cn02_doc_danh_sach.doc_danh_sach_dk import DieuKhienDocDanhSach
from chuc_nang.cn03_them_san_pham.them_san_pham_dk import DieuKhienThemSanPham
from chuc_nang.cn04_sua_san_pham.sua_san_pham_dk import DieuKhienSuaSanPham
from chuc_nang.cn05_xoa_san_pham.xoa_san_pham_dk import DieuKhienXoaSanPham
from chuc_nang.cn06_tim_kiem_sap_xep.tim_kiem_sap_xep_dk import DieuKhienTimKiemSapXep
from chuc_nang.cn07_thong_ke.thong_ke_dk import DieuKhienThongKe
from chuc_nang.cn08_hoan_tac_lam_lai.hoan_tac_lam_lai_dk import DieuKhienHoanTacLamLai
from chuc_nang.cn09_gio_hang.gio_hang_dk import DieuKhienGioHang


class DieuKhienCuaSoChinh:
    def __init__(self, root, quan_ly, nguoi_dung):
        self.root = root
        self.quan_ly = quan_ly
        self.nguoi_dung = nguoi_dung

        # 1. Tạo UI cửa sổ chính (vẽ widget)
        self.ui = CuaSoChinhUI(root, nguoi_dung.ho_ten, nguoi_dung.vai_tro)

        # 2. Tạo 8 dk con — mỗi cái nhận state từ quan_ly
        self.dk_danh_sach = DieuKhienDocDanhSach(self.ui.ui_bang, quan_ly.danh_sach)
        self.dk_them = DieuKhienThemSanPham(quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_sua = DieuKhienSuaSanPham(quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_xoa = DieuKhienXoaSanPham(quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_tim_sx = DieuKhienTimKiemSapXep(self.ui.ui_tim_sx, quan_ly.danh_sach, self.ui.ui_bang)
        self.dk_thong_ke = DieuKhienThongKe(self.ui.ui_thong_ke, quan_ly.danh_sach)
        self.dk_undoredo = DieuKhienHoanTacLamLai(self.ui.ui_undoredo, quan_ly.danh_sach, quan_ly.bo_undoredo)
        self.dk_gio_hang = DieuKhienGioHang(self.ui.ui_gio_hang, quan_ly.danh_sach, quan_ly.gio_hang)

        # 3. Gắn command cho 9 nút
        self._gan_su_kien()

        # 4. Nạp dữ liệu ban đầu
        self.refresh_all()

    def _gan_su_kien(self):
        # Nút Thêm → gọi hàm _them_san_pham của chính lớp này
        self.ui.btn_them.config(command=self._them_san_pham)
        self.ui.btn_sua.config(command=self._sua_san_pham)
        self.ui.btn_xoa.config(command=self._xoa_san_pham)
        self.ui.btn_dang_xuat.config(command=self._dang_xuat)

        # Double-click vào dòng bảng → mở form Sửa
        self.ui.bind_double_click_bang(self._sua_san_pham)

        # Thanh Tìm/Sắp xếp — dk con tự gắn sự kiện của nó
        self.dk_tim_sx.gan_su_kien()
        # Thanh Undo/Redo — dk con + truyền 2 callback
        self.dk_undoredo.gan_su_kien(
            cap_nhat_bang=lambda: self.dk_danh_sach.nap_lai(),
            cap_nhat_thong_ke=self.dk_thong_ke.cap_nhat
        )
        # Giỏ hàng — dk con + truyền callback lấy mã đang chọn
        self.dk_gio_hang.gan_su_kien(
            lay_ma_dang_chon_bang_chinh=self.ui.ui_bang.lay_ma_dang_chon
        )

    def refresh_all(self):
        """Sau mỗi thao tác (Thêm/Sửa/Xóa/Undo/Redo), gọi hàm này để refresh UI."""
        so_luong = self.dk_danh_sach.nap_lai()       # Nạp lại bảng
        self.dk_thong_ke.cap_nhat()                   # Nạp lại thống kê
        self.dk_gio_hang.cap_nhat_gio_hang()          # Nạp lại giỏ
        self.dk_undoredo.cap_nhat_trang_thai()        # Bật/tắt nút Undo/Redo
        self.ui.dat_trang_thai(f'Đã tải {so_luong} mặt hàng.')

    def _them_san_pham(self):
        """Khi user bấm nút Thêm → mở form → nếu lưu OK → refresh_all."""
        da_them = self.dk_them.mo_form_them(self.root)
        if da_them:
            self.refresh_all()  # ← đây là câu lệnh khiến bảng tự cập nhật sau khi Thêm

    def _sua_san_pham(self):
        ma = self.ui.ui_bang.lay_ma_dang_chon()
        if not ma:

    def _xoa_san_pham(self):

        if not ma:
            messagebox.showwarning('Chưa chọn', 'Vui lòng chọn 1 dòng để xóa.')
            return
        da_xoa = self.dk_xoa.xu_ly_xoa(self.root, ma, self.quan_ly.la_admin())
        if da_xoa:

10. 3 chế độ import: tĩnh, lazy, tương đối

Trong đồ án có 3 kiểu import. Hiểu rõ 3 kiểu này sẽ giúp bạn đọc code không bị rối:

10.1. Import tĩnh (ở đầu file)

Kiểu phổ biến nhất. Module được load ngay khi file chạy lần đầu.

# main.py
from quan_ly_trung_tam import QuanLyTrungTam
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_ui import CuaSoDangNhapUI
from khung_chinh.cua_so_chinh_dk import DieuKhienCuaSoChinh

10.2. Import tương đối (dấu chấm)

Khi file A và file B nằm cùng folder, A có thể import B mà không cần ghi đường dẫn đầy đủ. Dùng dấu chấm . (cùng folder) hoặc .. (folder cha).

# Trong cn01/dang_nhap_dk.py (cùng folder với dang_nhap_ui.py và dang_nhap_tt.py)
from .dang_nhap_ui import CuaSoDangNhapUI  # ← dấu chấm = cùng folder
from .dang_nhap_tt import BoDangNhap

Đây là lý do trong cua_so_chinh_dk.py có dòng from .cua_so_chinh_ui import CuaSoChinhUI — vì 2 file này cùng folder khung_chinh/.

10.3. Lazy import (bên trong hàm)

Một số module chỉ được import khi hàm được gọi. Lý do: tránh phụ thuộc vòng (A import B, B import A → lỗi).

# khung_chinh/khung_chinh_tt.py
class QuanLyTrungTam:
    def lay_danh_sach(self):
        # Lazy import: chỉ import khi hàm này được gọi
        from chuc_nang.cn02_doc_danh_sach.doc_danh_sach_tt import lay_danh_sach as _lay
        return _lay(self.danh_sach)

    def them_san_pham(self, du_lieu):
        from chuc_nang.cn03_them_san_pham.them_san_pham_tt import them_san_pham as _them
        return _them(self.danh_sach, self.bo_undoredo, du_lieu)

Nếu import tĩnh (ở đầu file), khung_chinh_tt.py sẽ import cn02_tt → cn02_tt import DSLK từ loi_thuat_toan → OK. Nhưng nếu cn02_tt lại cần QuanLyTrungTam thì sẽ bị vòng. Lazy import phá được vòng này.

📌 Tóm tắt 3 chế độ
Kiểu Cú pháp Khi nào dùng
Tĩnh from X.Y import Z Khi luôn cần module đó
Tương đối from .Y import Z Khi file cùng folder
Lazy def f(): from X import Y Tránh phụ thuộc vòng
