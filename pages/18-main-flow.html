📋 Bố cục cửa sổ chính — 4 vùng

  • Thanh đầu tiêu đề "QUẢN LÝ NHÀ SÁCH" bên trái + thông tin user ("Xin chào: admin (admin)") + nút Đăng xuất bên phải.
  • Thanh công cụ 1 hàng ngang gộp 3 nhóm: Thêm/Sửa/Xóa (cn03-05) + Undo/Redo (cn08) + Tìm/Sắp xếp (cn06), ngăn nhau bằng Separator dọc.
  • PanedWindow khung chính chia 2: trái (3 phần) là bảng sản phẩm (cn02), phải (2 phần) là Thống kê (cn07) + Giỏ hàng (cn09).
  • Thanh chân label trạng thái ("Sẵn sàng.") — sẽ đổi text khi user tương tác ("Đã xóa S001", "Đã thêm 1 món vào giỏ"...).
  • Nhân viên nếu user đăng nhập với vai trò nhan_vien thì nút Xóa bị ẩn (phân quyền cn01).
  • Mẹo user có thể double-click vào dòng trong bảng để mở form Sửa nhanh — không cần chọn rồi bấm nút Sửa.

Hình: CuaSoChinhUI(root, ho_ten, vai_tro) — tổng hợp 5 UI con (BangSanPhamUI + ThanhTimKiemSapXepUI + KhungThongKeUI + ThanhUndoRedoUI + KhungGioHangUI) vào 1 cửa sổ 1100×700.

1. main.py — điểm vào ứng dụng

import tkinter as tk

# Import state container
from quan_ly_trung_tam import QuanLyTrungTam
# Import UI + Điều khiển đăng nhập
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_ui import CuaSoDangNhapUI
from chuc_nang.cn01_dang_nhap_phan_quyen.dang_nhap_tt import BoDangNhap
# Import Điều khiển cửa sổ chính
from khung_chinh.cua_so_chinh_dk import DieuKhienCuaSoChinh


def main():
    """Hàm chính — khởi tạo ứng dụng."""
    # Khởi tạo state container — bộ não của ứng dụng
    qly = QuanLyTrungTam()
    if not qly.khoi_tao():
        print('LỖI: Không khởi tạo được hệ thống.')
        return

    # Tạo root Tk (cửa sổ chính của Tkinter)
    root = tk.Tk()
    root.title("Quản lý nhà sách")
    root.geometry("100x100")

    # Tạo bộ thuật toán đăng nhập
    bo_dn = BoDangNhap()

    # Hiện form đăng nhập modal
    form = CuaSoDangNhapUI(root)

    def _xu_ly_dang_nhap():
        """Hàm xử lý: đọc form → gọi bo_dn.dang_nhap → đóng form nếu OK."""
        ten = form.lay_ten_dang_nhap()
        mk = form.lay_mat_khau()
        if not ten or not mk:
            form.bao_loi('Vui lòng nhập đầy đủ tên đăng nhập và mật khẩu.')
            return
        ok, ket_qua = bo_dn.dang_nhap(ten, mk)
        if ok:
            form.nguoi_dung = ket_qua
            form.dong()
        else:
            form.bao_loi(ket_qua)
            form.xoa_mat_khau()

    # Gắn sự kiện
    form.entry_mk.bind('<Return>', lambda e: _xu_ly_dang_nhap())
    form.btn_dn.config(command=_xu_ly_dang_nhap)
    # Chờ đến khi form bị đóng
    root.wait_window(form.top)

    # Nếu đăng nhập thành công → mở cửa sổ chính
    if form.nguoi_dung:
        qly.nguoi_dung_hien_tai = form.nguoi_dung
        root.deiconify()
        app = DieuKhienCuaSoChinh(root, qly, form.nguoi_dung)
        root.mainloop()
    else:
        root.destroy()


if __name__ == '__main__':
    main()

Giải thích các dòng quan trọng

1.1. if __name__ == '__main__':

Đây là idiom Python — chỉ chạy main() khi file được chạy trực tiếp (python main.py), không chạy khi bị import từ file khác.

# Nếu chạy: python main.py
#   → __name__ của file main.py = '__main__'
#   → gọi main()

# Nếu trong file khác: import main
#   → __name__ của file main.py = 'main' (tên module)
#   → KHÔNG gọi main() — chỉ nạp các hàm/class để dùng

1.2. root.deiconify()

Hiện lại cửa sổ root sau khi đã withdraw() (ẩn). Trong đồ án, root được giữ nguyên (không ẩn) để Windows hiện icon dưới Taskbar. Nhưng trong cua_so_chinh_dk._dang_xuat() có dùng withdraw() + deiconify().

1.3. root.mainloop()

Vòng lặp chính của Tkinter — chặn luồng và lắng nghe sự kiện (click, gõ phím, v.v.) cho đến khi user đóng cửa sổ. Nếu không gọi mainloop(), cửa sổ hiện lên rồi tắt ngay.

2. QuanLyTrungTam — state container

Đây là "bộ não" của app — giữ các cấu trúc dữ liệu toàn cục:

import os

from loi_thuat_toan.danh_sach_lien_ket import DoublyLinkedList
from loi_thuat_toan.cau_truc_du_lieu import NganXep, HangDoi
from loi_thuat_toan.ket_noi_sqlite import (
    tao_bang_neu_chua_co,
    tai_du_lieu,
    tao_du_lieu_mau_neu_rong,
)
from loi_thuat_toan.nguoi_dung import (
    khoi_tao_nguoi_dung_mac_dinh,
    kiem_tra_dang_nhap,
)


class QuanLyTrungTam:
    """Lớp quản lý trung tâm — giữ cấu trúc dữ liệu toàn cục."""

    def __init__(self):
        # DSLK đôi chứa toàn bộ mặt hàng
        self.danh_sach = None
        # Dict chứa 2 Stack: 'undo' và 'redo'
        self.bo_undoredo = None
        # Queue giỏ hàng
        self.gio_hang = None
        # Người dùng đang đăng nhập (None = chưa login)
        self.nguoi_dung_hien_tai = None

4 thuộc tính toàn cục

Thuộc tính Kiểu Vai trò
danh_sach DoublyLinkedList Chứa toàn bộ sản phẩm (RAM)
bo_undoredo dict {'undo': NganXep, 'redo': NganXep} 2 Stack cho Undo/Redo
gio_hang HangDoi Queue FIFO cho giỏ hàng
nguoi_dung_hien_tai NguoiDung (hoặc None) Người dùng đang đăng nhập

3. Hàm khoi_tao() — khởi tạo hệ thống

def khoi_tao(self):
    """Tạo DB + bảng + dữ liệu mẫu + nạp vào DSLK + tạo Stack/Queue."""
    try:
        # Tạo bảng SQLite nếu chưa có
        tao_bang_neu_chua_co()
        # Nếu DB rỗng → chèn 5 sản phẩm mẫu
        tao_du_lieu_mau_neu_rong()
        # Tạo 2 tài khoản mặc định: admin/123 + nhanvien/123
        khoi_tao_nguoi_dung_mac_dinh()

        # Tạo DSLK đôi rỗng rồi tải dữ liệu từ SQLite vào
        self.danh_sach = DoublyLinkedList()
        tai_du_lieu(self.danh_sach)

        # 2 Stack cho Undo/Redo
        self.bo_undoredo = {
            'undo': NganXep(),
            'redo': NganXep()
        }
        # 1 Queue cho giỏ hàng
        self.gio_hang = HangDoi()

        print(f'[KHOI TAO] Da nap {self.danh_sach.so_luong} mat hang.')
        return True
    except Exception as loi:
        print(f'[LOI KHOI TAO] {loi}')
        return False

5 bước khởi tạo

  1. tao_bang_neu_chua_co() — tạo 2 bảng SQLite (mat_hang + nguoi_dung) nếu chưa có.
  2. tao_du_lieu_mau_neu_rong() — nếu bảng mat_hang rỗng → chèn 5 sản phẩm mẫu.
  3. khoi_tao_nguoi_dung_mac_dinh() — nếu bảng nguoi_dung rỗng → chèn admin + nhanvien.
  4. tai_du_lieu(self.danh_sach) — đọc tất cả sản phẩm từ SQLite → thêm vào DSLK.
  5. Khởi tạo 2 Stack + 1 Queue rỗng.
💡 Pattern "khởi tạo lười"

Các hàm tao_bang_neu_chua_co, tao_du_lieu_mau_neu_rong, khoi_tao_nguoi_dung_mac_dinh đều có chữ "nếu chưa có" — nghĩa là chỉ tạo khi cần.

Lần đầu chạy app: tạo mới. Lần sau: bỏ qua (vì đã có sẵn). Đảm bảo dữ liệu cũ không bị mất.

4. Pattern Facade (ủy quyền)

QuanLyTrungTam đóng vai trò Facade — mặt tiền. Mọi hàm của nó chỉ ủy quyền cho module chức năng con:

# QuanLyTrungTam.lay_danh_sach() — chỉ ủy quyền
def lay_danh_sach(self):
    from chuc_nang.cn02_doc_danh_sach.doc_danh_sach_tt import lay_danh_sach as _lay
    return _lay(self.danh_sach)

# QuanLyTrungTam.them_san_pham() — chỉ ủy quyền
def them_san_pham(self, du_lieu):
    from chuc_nang.cn03_them_san_pham.them_san_pham_tt import them_san_pham as _them
    return _them(self.danh_sach, self.bo_undoredo, du_lieu)

# QuanLyTrungTam.xoa_san_pham() — chỉ ủy quyền + kiểm tra quyền
def xoa_san_pham(self, ma_so):
    from chuc_nang.cn05_xoa_san_pham.xoa_san_pham_tt import xoa_san_pham as _xoa
    return _xoa(self.danh_sach, self.bo_undoredo, ma_so, self.la_admin())

Tại sao dùng Facade?

Vì test_app.py (47 test tự động) gọi API cũ của QuanLyTrungTam — nếu bỏ class này, tất cả test sẽ hỏng. Pattern Facade giúp:

  • Giữ API cũ: qly.them_san_pham(du_lieu) vẫn hoạt động.
  • Chuyển logic sang module con: dễ bảo trì, dễ test độc lập.
  • QuanLyTrungTam chỉ còn là "trung chuyển" — code ngắn, dễ đọc.

Lazy import

Lưu ý from ... import ... nằm bên trong hàm, không phải đầu file. Đây là lazy import — chỉ import khi hàm được gọi.

def lay_danh_sach(self):
    # Lazy import — chỉ chạy khi gọi hàm này
    from chuc_nang.cn02_doc_danh_sach.doc_danh_sach_tt import lay_danh_sach as _lay
    return _lay(self.danh_sach)

Lợi ích:

  • Tránh phụ thuộc vòng (circular import) — VD: cn02 import QuanLyTrungTam + QuanLyTrungTam import cn02.
  • Tăng tốc độ khởi động — không cần load tất cả module ngay từ đầu.

cd do_an_tkinter

python test_app.py

Ví dụ test mẫu

import pytest
from loi_thuat_toan.lop_doi_tuong import Sach, TapChi

def test_sach_tinh_gia_ban():
    """Test đa hình: Sach tính giá bán = gia × 1.1"""
    s = Sach('MH001', 'Truyện Kiều', 50000, 20, 'Nguyễn Du', 'NXB Văn học')
    assert s.tinh_gia_ban() == 55000   # 50000 × 1.1 = 55000
    assert s.loai_hang == 'Sách'

def test_tap_chi_tinh_gia_ban():
    """Test đa hình: TapChi tính giá bán = gia × 1.05"""
    tc = TapChi('MH002', 'Tạp chí KHCN', 30000, 15, 'Số 120')
    assert tc.tinh_gia_ban() == 31500   # 30000 × 1.05 = 31500

Cú pháp assert — nếu biểu thức sau assert là False, test fail. Nếu tất cả assert pass → test pass.
