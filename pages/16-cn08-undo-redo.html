↩️ CN08: Undo / Redo (Stack LIFO)

Đây là chức năng phức tạp nhất trong đồ án. Mỗi khi user thêm/sửa/xóa, hệ thống ghi 1 entry vào Stack Undo. Khi nhấn nút Undo, hệ thống lấy entry từ đỉnh Undo Stack, đảo ngược tác động, rồi đẩy sang Redo Stack. Nhấn Redo thì ngược lại.

📖 Mô tả yêu cầu barem

Nhóm 8 — Undo/Redo: Mọi thao tác thêm/sửa/xóa đều ghi lịch sử vào 2 Stack (LIFO). Undo = lấy từ đỉnh Undo Stack → đảo ngược → đẩy sang Redo. Redo = lấy từ đỉnh Redo Stack → áp dụng lại → đẩy sang Undo.

from loi_thuat_toan.lop_doi_tuong import Sach, TapChi, BaoGiay, LuanVan, BanThao

# HÀM TIỆN ÍCH NỘI BỘ — TÁI TẠO ĐỐI TƯỢNG TỪ DICT
# ============================================================
def _tao_doi_tuong_tu_dict(d):
    """Tái tạo đối tượng MatHang từ dict (dùng cho Undo thao tác xóa)."""
    loai = d.get('loai_hang')
    if loai == 'Sách':
        return Sach(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                    d['ton_kho'], d.get('tac_gia', ''), d.get('nha_xuat_ban', ''))
    elif loai == 'Tạp chí':
        return TapChi(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                      d['ton_kho'], d.get('so_phat_hanh', ''))
    elif loai == 'Báo giấy':
        return BaoGiay(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                       d['ton_kho'], d.get('ngay_xuat_ban', ''))
    elif loai == 'Luận văn':
        return LuanVan(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                       d['ton_kho'], d.get('tac_gia', ''),
                       d.get('truong_dai_hoc', ''), '')
    elif loai == 'Bản thảo':
        return BanThao(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                       d['ton_kho'], d.get('tac_gia', ''),
                       d.get('tinh_trang', ''))
    return None


# ============================================================
# HÀM UNDO — HOÀN TÁC THAO TÁC GẦN NHẤT
# ============================================================
def hoan_tac(danh_sach, bo_undoredo):
    """Undo — hoàn tác thao tác gần nhất."""
    try:
        # Lấy thao tác trên đỉnh Undo Stack (pop)
        thao_tac = bo_undoredo['undo'].lay_ra()
        if thao_tac is None:
            return False, 'Không có thao tác để hoàn tác.'

        # Đẩy sang Redo Stack (để có thể redo sau)
        bo_undoredo['redo'].day_vao(thao_tac)

        # Phân nhánh theo loại thao tác để đảo ngược tác động
        loai = thao_tac['loai']

        if loai == 'them':
            # Undo THÊM → XÓA sản phẩm vừa thêm
            ma = thao_tac['du_lieu_moi']['ma_so']
            danh_sach.xoa_node(ma)
            xoa_mot_san_pham(ma)
            return True, f'Đã hoàn tác: xóa "{thao_tac["du_lieu_moi"]["ten_san_pham"]}".'

        elif loai == 'xoa':
            # Undo XÓA → THÊM LẠI sản phẩm đã xóa
            cu = thao_tac['du_lieu_cu']
            sp = _tao_doi_tuong_tu_dict(cu)
            if sp:
                danh_sach.them_vao_cuoi(sp)
                luu_mot_san_pham(sp)
            return True, f'Đã hoàn tác: thêm lại "{cu["ten_san_pham"]}".'

        elif loai == 'sua':
            # Undo SỬA → đặt lại dữ liệu cũ
            cu = thao_tac['du_lieu_cu']
            ma = cu['ma_so']
            dl = {
                'ten_san_pham': cu['ten_san_pham'],
                'gia_co_ban':   cu['gia_co_ban'],
                'ton_kho':      cu['ton_kho']
            }
            # Thêm các trường riêng nếu có
            for k in ['tac_gia', 'nha_xuat_ban', 'so_phat_hanh',
                      'ngay_xuat_ban', 'truong_dai_hoc', 'tinh_trang']:
                if k in cu:
                    dl[k] = cu[k]
            sp = danh_sach.tim_theo_ma_so(ma)
            if sp:
                danh_sach.cap_nhat_node(ma, dl)
                cap_nhat_san_pham(sp)
            return True, f'Đã hoàn tác: khôi phục "{cu["ten_san_pham"]}".'

        return False, 'Thao tác không xác định.'
    except Exception as loi:
        return False, f'Lỗi: {loi}'


# ============================================================
# HÀM REDO — LÀM LẠI THAO TÁC VỪA UNDO
# ============================================================
def lam_lai(danh_sach, bo_undoredo):
    """Redo — làm lại thao tác đã undo."""
    try:
        # Lấy thao tác trên đỉnh Redo Stack (pop)
        thao_tac = bo_undoredo['redo'].lay_ra()
        if thao_tac is None:
            return False, 'Không có thao tác để làm lại.'

        # Đẩy lại sang Undo Stack
        bo_undoredo['undo'].day_vao(thao_tac)

        loai = thao_tac['loai']

        if loai == 'them':
            # Redo THÊM → THÊM LẠI sản phẩm
            moi = thao_tac['du_lieu_moi']
            sp = _tao_doi_tuong_tu_dict(moi)
            if sp:
                danh_sach.them_vao_cuoi(sp)
                luu_mot_san_pham(sp)
            return True, f'Đã làm lại: thêm "{moi["ten_san_pham"]}".'

        elif loai == 'xoa':
            # Redo XÓA → XÓA LẠI sản phẩm
            ma = thao_tac['du_lieu_cu']['ma_so']
            danh_sach.xoa_node(ma)
            xoa_mot_san_pham(ma)
            return True, f'Đã làm lại: xóa "{thao_tac["du_lieu_cu"]["ten_san_pham"]}".'

        elif loai == 'sua':
            # Redo SỬA → áp dụng lại dữ liệu mới
            moi = thao_tac['du_lieu_moi']
            ma = moi['ma_so']
            dl = {
                'ten_san_pham': moi['ten_san_pham'],
                'gia_co_ban':   moi['gia_co_ban'],
                'ton_kho':      moi['ton_kho']
            }
            for k in ['tac_gia', 'nha_xuat_ban', 'so_phat_hanh',
                      'ngay_xuat_ban', 'truong_dai_hoc', 'tinh_trang']:
                if k in moi:
                    dl[k] = moi[k]
            sp = danh_sach.tim_theo_ma_so(ma)
            if sp:
                danh_sach.cap_nhat_node(ma, dl)
                cap_nhat_san_pham(sp)
            return True, f'Đã làm lại: cập nhật "{moi["ten_san_pham"]}".'

        return False, 'Thao tác không xác định.'
    except Exception as loi:
        return False, f'Lỗi: {loi}'


# ============================================================
# HÀM TRẠNG THÁI — CHO UI BẬT/TẮT NÚT
# ============================================================
def trang_thai(bo_undoredo):
    """Trả về dict trạng thái Undo/Redo để UI bật/tắt nút."""
    return {
        'co_the_undo': not bo_undoredo['undo'].rong(),
        'co_the_redo': not bo_undoredo['redo'].rong(),
        'so_undo':     bo_undoredo['undo'].so_luong,
        'so_redo':     bo_undoredo['redo'].so_luong,
    }

3. Đảo ngược 3 loại thao tác

Đây là phần phức tạp nhất — mỗi loại thao tác (them/sua/xoa) có cách đảo ngược khác nhau:

Thao tác gốc Undo (đảo ngược) Redo (áp dụng lại)
them (thêm mới) Xóa sản phẩm vừa thêm Thêm lại sản phẩm
sua (cập nhật) Khôi phục dữ liệu cũ Áp dụng lại dữ liệu mới
xoa (xóa) Thêm lại sản phẩm đã xóa Xóa lại sản phẩm

Ví dụ cụ thể: Undo thao tác Thêm

# Giả sử user vừa thêm sản phẩm "Sách Python" (mã MH099)
# Undo entry đã được ghi:
thao_tac = {
    'loai': 'them',
    'du_lieu_cu': None,
    'du_lieu_moi': {
        'ma_so': 'MH099',
        'ten_san_pham': 'Sách Python',
        'loai_hang': 'Sách',
        'gia_co_ban': 80000,
        'ton_kho': 10,
        ...
    }
}

# Khi nhấn Undo:
# 1. Lấy thao_tac ra khỏi Undo Stack (pop)
# 2. Đẩy sang Redo Stack
# 3. Vì loai == 'them' → đảo ngược = XÓA
#    → ma = thao_tac['du_lieu_moi']['ma_so'] = 'MH099'
#    → gọi danh_sach.xoa_node('MH099') + xoa_mot_san_pham('MH099')

Ví dụ cụ thể: Undo thao tác Xóa

# Giả sử user vừa xóa "Truyện Kiều" (mã MH001)
thao_tac = {
    'loai': 'xoa',
    'du_lieu_cu': {
        'ma_so': 'MH001',
        'ten_san_pham': 'Truyện Kiều',
        'loai_hang': 'Sách',
        'gia_co_ban': 50000,
        'ton_kho': 20,
        'tac_gia': 'Nguyễn Du',
        'nha_xuat_ban': 'NXB Văn học'
    },
    'du_lieu_moi': None
}

# Khi nhấn Undo:
# 1. Pop khỏi Undo Stack → đẩy sang Redo Stack
# 2. Vì loai == 'xoa' → đảo ngược = THÊM LẠI
#    → cu = thao_tac['du_lieu_cu']
#    → sp = _tao_doi_tuong_tu_dict(cu)   # Tái tạo đối tượng Sach
#    → gọi danh_sach.them_vao_cuoi(sp) + luu_mot_san_pham(sp)

📋 Ý nghĩa 2 nút + 3 trạng thái

  • ↶ Hoàn tác (Undo) — đảo ngược thao tác cuối cùng (Thêm → Xóa, Xóa → Thêm lại, Sửa → trả về giá cũ).
  • ↷ Làm lại (Redo) — áp dụng lại thao tác vừa bị Undo (Xóa lại cái vừa được khôi phục).
  • Stack Undo lưu lịch sử thao tác theo LIFO — pop() phần tử cuối khi bấm Undo.
  • Stack Redo lưu thao tác bị Undo — khi bấm Undo thì đẩy thao tác đó sang Redo; khi bấm Redo thì pop Redo → chạy lại → đẩy về Undo.
  • 3 trạng thái tùy theo 2 stack rỗng hay có phần tử mà 2 nút được bật/tắt tương ứng qua bat_hoan_tac(True/False) + bat_lam_lai(True/False).
  • Lưu ý nếu user thực hiện thao tác mới sau khi Undo ⇒ stack Redo bị clear (theo quy ước chung).

Hình: ThanhUndoRedoUI(parent) — chỉ 2 nút nhưng trạng thái enabled/disabled thay đổi liên tục theo 2 stack LIFO.

from tkinter import ttk


class ThanhUndoRedoUI:
    """Thanh chứa 2 nút Undo + Redo."""

    def __init__(self, parent):
        # Nút Undo — command sẽ gắn bởi Điều khiển
        self.btn_hoan_tac = ttk.Button(parent, text='↶ Hoàn tác', width=12)
        self.btn_hoan_tac.pack(side='left', padx=2)

        # Nút Redo
        self.btn_lam_lai = ttk.Button(parent, text='↷ Làm lại', width=12)
        self.btn_lam_lai.pack(side='left', padx=2)

    def bat_hoan_tac(self, bat=True):
        """Bật/tắt nút Undo."""
        self.btn_hoan_tac.config(state='normal' if bat else 'disabled')

    def bat_lam_lai(self, bat=True):
        """Bật/tắt nút Redo."""
        self.btn_lam_lai.config(state='normal' if bat else 'disabled')

    def vo_hieu_hoa_tat_ca(self):
        """Vô hiệu hóa cả 2 nút (dùng khi chưa có thao tác nào)."""
        self.bat_hoan_tac(False)
        self.bat_lam_lai(False)

State 'normal' vs 'disabled'

Button có 3 trạng thái:

  • 'normal': bình thường — user click được.
  • 'disabled': vô hiệu hóa — xám, không click được.
  • 'active': khi di chuột qua (ít dùng).

Trong đồ án: nút Undo bị disabled khi Stack Undo rỗng, nút Redo bị disabled khi Stack Redo rỗng — tránh user click không có tác dụng.

5. File _dk: DieuKhienHoanTacLamLai

from .hoan_tac_lam_lai_tt import hoan_tac, lam_lai, trang_thai


class DieuKhienHoanTacLamLai:
    def __init__(self, ui_undoredo, danh_sach, bo_undoredo):
        self.ui = ui_undoredo
        self.danh_sach = danh_sach
        self.bo_undoredo = bo_undoredo

    def gan_su_kien(self, cap_nhat_bang, cap_nhat_thong_ke):
        """
        Gắn command cho 2 nút + lưu 2 callback để gọi sau undo/redo.

        Tham số:
            cap_nhat_bang       : hàm callback để nạp lại bảng sau undo/redo.
            cap_nhat_thong_ke   : hàm callback để cập nhật thống kê.
        """
        self._cap_nhat_bang = cap_nhat_bang
        self._cap_nhat_thong_ke = cap_nhat_thong_ke

        self.ui.btn_hoan_tac.config(command=self.xu_ly_hoan_tac)
        self.ui.btn_lam_lai.config(command=self.xu_ly_lam_lai)

    def xu_ly_hoan_tac(self):
        """Gọi thuật toán Undo → cập nhật UI."""
        ok, msg = hoan_tac(self.danh_sach, self.bo_undoredo)
        self.cap_nhat_trang_thai()
        if ok:
            # Sau undo → nạp lại bảng + thống kê
            self._cap_nhat_bang()
            self._cap_nhat_thong_ke()
        return ok, msg

    def xu_ly_lam_lai(self):
        """Gọi thuật toán Redo → cập nhật UI."""
        ok, msg = lam_lai(self.danh_sach, self.bo_undoredo)
        self.cap_nhat_trang_thai()
        if ok:
            self._cap_nhat_bang()
            self._cap_nhat_thong_ke()
        return ok, msg

    def cap_nhat_trang_thai(self):
        """Lấy trạng thái Stack → bật/tắt 2 nút."""
        tt = trang_thai(self.bo_undoredo)
        self.ui.bat_hoan_tac(tt['co_the_undo'])
        self.ui.bat_lam_lai(tt['co_the_redo'])

6. Cơ chế callback sau Undo/Redo

Đây là pattern quan trọng — Điều khiển Undo/Redo cần gọi lại 2 hàm sau khi hoàn tất:

  1. cap_nhat_bang(): nạp lại bảng sản phẩm (vì dữ liệu đã thay đổi).
  2. cap_nhat_thong_ke(): cập nhật khung thống kê.

Vì DieuKhienHoanTacLamLai không biết về bảng + thống kê (chỉ biết về Stack), nên nó nhận 2 callback từ caller:

# Trong DieuKhienCuaSoChinh._gan_su_kien
self.dk_undoredo.gan_su_kien(
    cap_nhat_bang=lambda: self.dk_danh_sach.nap_lai(),
    cap_nhat_thong_ke=self.dk_thong_ke.cap_nhat
)

2 cách truyền callback

# Cách 1: Truyền hàm trực tiếp (nếu tên hàm + chữ ký khớp)
self.dk_undoredo.gan_su_kien(
    cap_nhat_bang=self.dk_danh_sach.nap_lai,        # nap_lai() không tham số → OK
    cap_nhat_thong_ke=self.dk_thong_ke.cap_nhat     # cap_nhat() không tham số → OK
)

# Cách 2: Dùng lambda để bọc (linh hoạt hơn)
self.dk_undoredo.gan_su_kien(
    cap_nhat_bang=lambda: self.dk_danh_sach.nap_lai(),
    cap_nhat_thong_ke=lambda: self.dk_thong_ke.cap_nhat()
)

Cách 1 gọn hơn, cách 2 linh hoạt hơn (có thể thêm logic khác vào lambda).

Vòng đời Undo/Redo — ví dụ đầy đủ

Giả sử user thực hiện 3 thao tác: Thêm A, Sửa A, Xóa A. Sau đó nhấn Undo 2 lần, Redo 1 lần:

Vòng đời Undo/Redo

1. Thêm A → Undo Stack: [Thêm A]   Redo Stack: []

2. Sửa A → Undo Stack: [Thêm A, Sửa A]   Redo Stack: []

3. Xóa A → Undo Stack: [Thêm A, Sửa A, Xóa A]   Redo Stack: []

↓ Nhấn Undo lần 1 (hoàn tác Xóa A → A xuất hiện lại)

4. Undo Stack: [Thêm A, Sửa A]   Redo Stack: [Xóa A]

↓ Nhấn Undo lần 2 (hoàn tác Sửa A → A về dữ liệu cũ)

5. Undo Stack: [Thêm A]   Redo Stack: [Xóa A, Sửa A]

↓ Nhấn Redo (làm lại Sửa A → A về dữ liệu mới)

6. Undo Stack: [Thêm A, Sửa A]   Redo Stack: [Xóa A]

⚠️ Quy tắc "Xóa Redo khi có thao tác mới"

Sau bước 5, nếu user thực hiện thao tác mới (VD: Thêm B) thay vì Redo, thì toàn bộ Redo Stack [Xóa A, Sửa A] sẽ bị xóa:

Undo Stack: [Thêm A, Sửa A, Thêm B]   Redo Stack: []

Lý do: lịch sử đã "rẽ nhánh" — Sửa A và Xóa A không còn hợp lệ trong nhánh mới.

✅ Tóm tắt CN08
  • 2 Stack: undo (hoàn tác) + redo (làm lại).
  • Undo: pop từ Undo Stack → đảo ngược → push sang Redo Stack.
  • Redo: pop từ Redo Stack → áp dụng lại → push sang Undo Stack.
  • 3 loại thao tác: them/sua/xoa — mỗi loại có cách đảo ngược riêng.
  • UI: 2 nút, disabled khi Stack tương ứng rỗng.
  • Sau Undo/Redo, gọi callback để cập nhật bảng + thống kê.
