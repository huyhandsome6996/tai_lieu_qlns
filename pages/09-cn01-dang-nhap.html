🔐 CN01: Đăng nhập + Phân quyền

Đây là chức năng đầu tiên người dùng gặp khi mở app. Hệ thống yêu cầu đăng nhập bằng tên đăng nhập + mật khẩu. Sau khi đăng nhập thành công, hệ thống kiểm tra vai trò (admin hoặc nhân viên) để quyết định quyền hạn.

📖 Mô tả yêu cầu barem

Nhóm 1 — Đăng nhập + Phân quyền: Người dùng nhập tên/mật khẩu → hệ thống kiểm tra trong SQLite → nếu đúng thì mở cửa sổ chính; nếu là admin thì toàn quyền, nếu là nhân viên thì bị hạn chế (không được xóa sản phẩm).

2. File _tt: BoDangNhap (thuật toán)

File này chỉ chứa logic — không import tkinter. Có 3 hàm chính:

  • dang_nhap(ten, mk): kiểm tra tên/mật khẩu với SQLite.
  • dang_xuat(): reset người dùng hiện tại về None.
  • la_admin(): kiểm tra vai trò admin.
# Import hàm kiểm tra đăng nhập từ thư viện thuật toán (loi_thuat_toan/).
from loi_thuat_toan.nguoi_dung import kiem_tra_dang_nhap


class BoDangNhap:
    """
    Bộ thuật toán đăng nhập.
    Giữ người dùng đang đăng nhập hiện tại + cung cấp hàm tiện ích kiểm tra quyền.
    """

    def __init__(self):
        """Khởi tạo: chưa có người dùng nào đăng nhập."""
        # nguoi_dung_hien_tai = None nghĩa là chưa login.
        self.nguoi_dung_hien_tai = None

    def dang_nhap(self, ten_dang_nhap, mat_khau):
        """
        Kiểm tra đăng nhập.

        Tham số:
            ten_dang_nhap : str — tên đăng nhập user nhập.
            mat_khau      : str — mật khẩu user nhập.

        Trả về:
            (True,  nguoi_dung) nếu đăng nhập đúng.
            (False, thong_bao)  nếu sai tên/mật khẩu hoặc lỗi.
        """
        try:
            # Gọi hàm kiểm tra từ thư viện (đã bọc try/except ở tầng dưới).
            nguoi_dung = kiem_tra_dang_nhap(ten_dang_nhap, mat_khau)
            # Nếu hàm trả về None → sai tên hoặc mật khẩu.
            if nguoi_dung is None:
                return False, 'Sai tên đăng nhập hoặc mật khẩu.'
            # Lưu người dùng hiện tại vào bộ.
            self.nguoi_dung_hien_tai = nguoi_dung
            return True, nguoi_dung
        except Exception as loi:
            # Bất kỳ lỗi nào khác → trả về tuple (False, thông báo lỗi).
            return False, f'Lỗi: {loi}'

    def dang_xuat(self):
        """Đăng xuất — reset người dùng hiện tại về None."""
        self.nguoi_dung_hien_tai = None

    def la_admin(self):
        """Kiểm tra người dùng hiện tại có phải admin không."""
        return (self.nguoi_dung_hien_tai is not None and
                self.nguoi_dung_hien_tai.vai_tro == 'admin')

Giải thích từng dòng quan trọng

Hàm kiem_tra_dang_nhap được import từ loi_thuat_toan/nguoi_dung.py — file này đã được học ở bài "Quản lý người dùng". Hàm đó sẽ:

  1. Kết nối SQLite, tìm user theo ten_dang_nhap.
  2. Nếu không thấy → trả về None.
  3. Nếu thấy → so sánh mật khẩu nhập với mật khẩu DB.
  4. Nếu khớp → trả về đối tượng NguoiDung.
  5. Nếu không khớp → trả về None.

Kiểu trả về của dang_nhap là tuple 2 phần tử:

  • Nếu thành công: (True, nguoi_dung)
  • Nếu thất bại: (False, 'thông báo lỗi')

Kiểu trả về "tuple 2 giá trị" này rất phổ biến trong đồ án — caller chỉ cần kiểm tra ok để biết thành công hay không:

ok, ket_qua = bo.dang_nhap('admin', '123')
if ok:
    print('Đăng nhập thành công:', ket_qua.ho_ten)
else:
    print('Lỗi:', ket_qua)   # ket_qua lúc này là chuỗi thông báo lỗi

File này chỉ chứa code Tkinter — vẽ form đăng nhập. Form này modal (chặn tương tác cửa sổ cha cho đến khi đóng).

import tkinter as tk
from tkinter import ttk, messagebox


class CuaSoDangNhapUI:
    """Cửa sổ giao diện đăng nhập — chạy modal."""

    def __init__(self, parent):
        # Tạo Toplevel (cửa sổ con) — modal.
        self.top = tk.Toplevel(parent)
        self.top.title('Đăng nhập hệ thống')
        self.top.geometry('380x220+500+250')
        # Làm cửa sổ modal: chặn tương tác parent.
        self.top.transient(parent)
        self.top.grab_set()

        # Ép cửa sổ nổi lên trên cùng (đè lên VS Code) để dễ nhìn thấy.
        self.top.lift()
        self.top.attributes('-topmost', True)
        # Bỏ topmost đi ngay lập tức để sau này không bị kẹt dính trên cùng.
        self.top.after(100, lambda: self.top.attributes('-topmost', False))
        self.top.focus_force()

        # Ban đầu chưa có người dùng — sẽ được gán nếu login OK.
        self.nguoi_dung = None

        # Gọi hàm tạo giao diện.
        self._tao_giao_dien()

    def _tao_giao_dien(self):
        """Tạo các widget (label, entry, button) cho form đăng nhập."""
        # Frame chứa nội dung, padding 20px các phía.
        frame = ttk.Frame(self.top, padding=20)
        frame.pack(fill='both', expand=True)

        # Tiêu đề.
        ttk.Label(frame, text='ĐĂNG NHẬP',
                  font=('Arial', 16, 'bold')).grid(row=0, column=0, columnspan=2, pady=(0, 15))

        # Label + Entry cho tên đăng nhập.
        ttk.Label(frame, text='Tên đăng nhập:').grid(row=1, column=0, sticky='e', padx=5, pady=5)
        self.entry_ten = ttk.Entry(frame, width=25)
        self.entry_ten.grid(row=1, column=1, padx=5, pady=5)
        self.entry_ten.focus()  # Focus vào ô này khi mở form.

        # Label + Entry cho mật khẩu (show='*' để ẩn ký tự).
        ttk.Label(frame, text='Mật khẩu:').grid(row=2, column=0, sticky='e', padx=5, pady=5)
        self.entry_mk = ttk.Entry(frame, width=25, show='*')
        self.entry_mk.grid(row=2, column=1, padx=5, pady=5)

        # Bind phím Enter → chuyển focus sang ô mật khẩu (nếu đang ở ô tên).
        self.entry_ten.bind('<Return>', lambda e: self.entry_mk.focus())

        # Nút đăng nhập (columnspan=2 để rộng qua 2 cột).
        self.btn_dn = ttk.Button(frame, text='Đăng nhập')
        self.btn_dn.grid(row=3, column=0, columnspan=2, pady=15, sticky='ew')

        # Gợi ý tài khoản mẫu.
        ttk.Label(frame, text='(Mặc định: admin/123 hoặc nhanvien/123)',
                  foreground='gray').grid(row=4, column=0, columnspan=2)

    def lay_ten_dang_nhap(self):
        """Trả về tên đăng nhập user nhập (đã strip khoảng trắng)."""
        return self.entry_ten.get().strip()

    def lay_mat_khau(self):
        """Trả về mật khẩu user nhập (đã strip khoảng trắng)."""
        return self.entry_mk.get().strip()

    def xoa_mat_khau(self):
        """Xóa ô mật khẩu — dùng khi sai mật khẩu để user nhập lại."""
        self.entry_mk.delete(0, tk.END)

    def bao_loi(self, thong_bao):
        """Hiển thị hộp thoại lỗi."""
        messagebox.showerror('Đăng nhập thất bại', thong_bao, parent=self.top)

    def dong(self):
        """Đóng cửa sổ đăng nhập."""
        self.top.destroy()

Giải thích các cú pháp Tkinter mới

3.1. tk.Toplevel(parent)

Tạo cửa sổ con độc lập với cửa sổ cha (parent). Khác với Frame (nhúng trong parent), Toplevel có thanh tiêu đề riêng, có thể di chuyển độc lập.

3.2. transient(parent) + grab_set()

Hai lệnh này biến cửa sổ thành modal:

  • transient(parent): cửa sổ con luôn nằm trên parent, khi parent bị ẩn thì con cũng ẩn theo.
  • grab_set(): chặn mọi tương tác với parent — user bắt buộc phải đóng form đăng nhập mới thao tác được với cửa sổ chính.

3.3. attributes('-topmost', True)

Ép cửa sổ nổi lên trên cùng của hệ điều hành — đè cả VS Code, Chrome, v.v. Dùng khi mở form để đảm bảo user nhìn thấy ngay. Sau 100ms thì tắt để không bị "dính" trên cùng.

self.top.attributes('-topmost', True)
# Tắt topmost sau 100ms để không bị kẹt
self.top.after(100, lambda: self.top.attributes('-topmost', False))

3.4. show='*' trong Entry

Ẩn ký tự nhập (cho mật khẩu). Mỗi ký tự user nhập sẽ hiển thị là dấu *.

3.5. bind('<Return>', callback)

Gắn phím Enter cho widget. Khi user nhấn Enter trong widget này, callback sẽ được gọi. Callback phải nhận 1 tham số event (dùng lambda e: ... để bỏ qua).

3.6. columnspan=2 trong grid

Mặc định 1 widget chiếm 1 cột. columnspan=2 cho phép widget trải dài qua 2 cột. Dùng cho nút Đăng nhập để rộng vừa khít form.

3.7. sticky='ew'

Kéo dãn widget để lấp đầy cell theo cả chiều East + West (trái + phải). Có 4 giá trị:

  • 'n': top (North)
  • 's': bottom (South)
  • 'e': right (East)
  • 'w': left (West)
  • 'ew': lấp ngang
  • 'ns': lấp dọc
  • 'nsew': lấp cả 4 chiều

4. File _dk: DieuKhienDangNhap (điều khiển)

Lớp điều khiển đóng vai trò cầu nối giữa UI và Thuật toán:

  • UI chỉ quan tâm tới giao diện (vẽ form, bắt sự kiện).
  • Thuật toán chỉ quan tâm tới logic (kiểm tra tên/mật khẩu).
  • Điều khiển: gắn sự kiện UI → gọi thuật toán → xử lý kết quả → cập nhật UI.
# Import giao diện (UI) của chức năng đăng nhập.
from .dang_nhap_ui import CuaSoDangNhapUI
# Import thuật toán (logic) của chức năng đăng nhập.
from .dang_nhap_tt import BoDangNhap


class DieuKhienDangNhap:
    """Lớp điều khiển — quản lý luồng đăng nhập / đăng xuất của ứng dụng."""

    def __init__(self):
        """Khởi tạo bộ điều khiển + bộ thuật toán đăng nhập."""
        # Tạo 1 đối tượng BoDangNhap (logic) — dùng chung cho toàn app.
        self.bo = BoDangNhap()

    def hien_form_dang_nhap(self, parent):
        """
        Hiện form đăng nhập modal. Chờ user nhập → kiểm tra → đóng form.

        Trả về:
            NguoiDung nếu đăng nhập thành công, None nếu user đóng form.
        """
        # Tạo đối tượng UI (form đăng nhập).
        form = CuaSoDangNhapUI(parent)

        # Gắn sự kiện: phím Enter ở ô mật khẩu + nút "Đăng nhập" → xử lý đăng nhập.
        form.entry_mk.bind('<Return>', lambda e: self._xu_ly_dang_nhap(form))
        form.btn_dn.config(command=lambda: self._xu_ly_dang_nhap(form))

        # Chờ đến khi form bị đóng (bởi nút đăng nhập thành công hoặc user tắt).
        parent.wait_window(form.top)

        # Trả về người dùng (None nếu user tắt form mà không đăng nhập).
        return form.nguoi_dung

    def _xu_ly_dang_nhap(self, form):
        """Lấy dữ liệu từ form → gọi thuật toán → xử lý kết quả."""
        # Lấy tên đăng nhập + mật khẩu từ form.
        ten = form.lay_ten_dang_nhap()
        mk = form.lay_mat_khau()

        # Validate cơ bản: không được để trống.
        if not ten or not mk:
            form.bao_loi('Vui lòng nhập đầy đủ tên đăng nhập và mật khẩu.')
            return

        # Gọi hàm thuật toán đăng nhập.
        ok, ket_qua = self.bo.dang_nhap(ten, mk)
        if ok:
            # Nếu OK → lưu người dùng vào form (để parent đọc) → đóng form.
            form.nguoi_dung = ket_qua
            form.dong()
        else:
            # Nếu sai → báo lỗi + xóa ô mật khẩu để user nhập lại.
            form.bao_loi(ket_qua)
            form.xoa_mat_khau()

    def dang_xuat(self):
        """Đăng xuất — gọi hàm đăng xuất của bộ thuật toán."""
        self.bo.dang_xuat()

    def la_admin(self):
        """Trả về True nếu người dùng hiện tại là admin."""
        return self.bo.la_admin()

    @property
    def nguoi_dung_hien_tai(self):
        """Trả về người dùng đang đăng nhập hiện tại (None nếu chưa login)."""
        return self.bo.nguoi_dung_hien_tai

Giải thích các cú pháp Python mới

4.1. Import相对 from .dang_nhap_ui import ...

Dấu chấm . trước tên module nghĩa là import từ cùng folder. .dang_nhap_ui = file dang_nhap_ui.py trong cùng folder với file hiện tại.

4.2. lambda e: ...

Hàm ẩn danh (anonymous function). Dùng khi cần 1 hàm nhỏ, không muốn đặt tên. e là tham số bắt buộc khi bind sự kiện (Tkinter luôn truyền event object vào callback).

# Cách 1: Định nghĩa hàm bình thường
def xu_ly(event):
    self._xu_ly_dang_nhap(form)
form.btn_dn.bind('<Return>', xu_ly)

# Cách 2: Dùng lambda — gọn hơn
form.btn_dn.bind('<Return>', lambda e: self._xu_ly_dang_nhap(form))

4.3. @property

Decorator biến một method thành thuộc tính — gọi như biến, không cần dấu ngoặc.

@property
def nguoi_dung_hien_tai(self):
    return self.bo.nguoi_dung_hien_tai

# Cách dùng:
dk = DieuKhienDangNhap()
user = dk.nguoi_dung_hien_tai   # KHÔNG có dấu ngoặc ()
# Tương đương với: user = dk.nguoi_dung_hien_tai()

4.4. parent.wait_window(form.top)

Block luồng chính cho đến khi cửa sổ form.top bị đóng. Đây là cơ chế modal: code sau dòng này chỉ chạy khi user đã đóng form.

form = CuaSoDangNhapUI(parent)
# ... gắn sự kiện ...
parent.wait_window(form.top)        # ← DỪNG Ở ĐÂY cho đến khi form đóng
# Code dưới đây chỉ chạy sau khi form đóng
return form.nguoi_dung              # Đọc dữ liệu user đã nhập

6. Cơ chế phân quyền Admin / Nhân viên

Phân quyền dựa vào trường vai_tro trong bảng SQLite bang_nguoi_dung:

Vai trò ten_dang_nhap mat_khau Quyền
admin admin 123 Toàn quyền: xem, thêm, sửa, xóa, giỏ hàng
nhan_vien nhanvien 123 Hạn chế: xem, thêm, sửa, giỏ hàng — KHÔNG được xóa

Cách kiểm tra quyền trong code

Ở cua_so_chinh_dk.py, sau khi đăng nhập thành công, nếu user là nhân viên thì nút Xóa bị vô hiệu hóa:

# Nếu là nhân viên → ẩn nút Xóa (vì không được phép xóa).
if nguoi_dung.vai_tro != 'admin':
    self.ui.btn_xoa.config(state='disabled')

Và ở tầng thuật toán (xoa_san_pham_tt.py), có thêm lớp bảo vệ thứ hai — kể cả nếu somehow gọi được hàm xóa, nó vẫn kiểm tra:

        return False, 'BẠN LÀ NHÂN VIÊN - KHÔNG ĐƯỢC PHÉP XÓA SẢN PHẨM.'
    # ... tiếp tục logic xóa ...
🔒 Bảo vệ 2 lớp

Phân quyền được bảo vệ ở 2 lớp độc lập:

  1. Lớp UI: vô hiệu nút Xóa → nhân viên không click được.
  2. Lớp thuật toán: kiểm tra la_admin trước khi xóa → kể cả nếu gọi trực tiếp hàm xoa_san_pham, nó vẫn từ chối.

Đây là best practice trong bảo mật — defense in depth (phòng thủ nhiều lớp).
