🔀 Bài 6: Merge Sort (Thuật toán sắp xếp trộn)

Merge Sort là thuật toán sắp xếp O(n log n) — nhanh hơn Bubble Sort, Insertion Sort. Đây là phần khó nhất về thuật toán trong đồ án, do Huy phụ trách.

🚨 Đọc chậm + vẽ ra giấy

Merge Sort dùng đệ quy — khái niệm khó hiểu nếu mới lần đầu. Hãy lấy giấy bút ra vẽ theo từng bước trong bài.

1. Ý tưởng Merge Sort

Merge Sort dựa trên nguyên lý "chia để trị" (divide and conquer):

  1. Chia: cắt danh sách thành 2 nửa.
  2. Trị: đệ quy sắp xếp từng nửa.
  3. Trộn: ghép 2 nửa đã sắp xếp thành 1 danh sách cũng sắp xếp.
Sơ đồ Merge Sort
            [38, 27, 43, 3, 9, 82, 10]
                    ↓ CHIA
        [38, 27, 43, 3]    [9, 82, 10]
              ↓ CHIA           ↓ CHIA
      [38, 27]  [43, 3]   [9, 82]  [10]
        ↓ CHIA    ↓ CHIA    ↓         |
     [38][27]  [43][3]   [9][82]    [10]
        ↓ TRỘN    ↓ TRỘN    ↓ TRỘN    |
      [27, 38]  [3, 43]   [9, 82]    [10]
          ↓ TRỘN              ↓ TRỘN
       [3, 27, 38, 43]    [9, 10, 82]
                ↓ TRỘN
         [3, 9, 10, 27, 38, 43, 82]

2. Đệ quy là gì?

Đệ quy (recursion) = hàm tự gọi lại chính nó. Mỗi lần gọi với input nhỏ hơn, đến khi đạt điều kiện dừng thì trả về.

# Ví dụ: tính giai thừa n! = n × (n-1) × ... × 1
# Cách 1: vòng lặp
def giai_thua_lap(n):
    ket_qua = 1
    for i in range(1, n + 1):
        ket_qua *= i
    return ket_qua

# Cách 2: đệ quy
def giai_thua_de_quy(n):
    # Điều kiện dừng — BẮT BUỘC phải có!
    if n <= 1:
        return 1
    # Tự gọi lại với input nhỏ hơn
    return n * giai_thua_de_quy(n - 1)

print(giai_thua_de_quy(5))
# 5 * giai_thua_de_quy(4)
# = 5 * 4 * giai_thua_de_quy(3)
# = 5 * 4 * 3 * giai_thua_de_quy(2)
# = 5 * 4 * 3 * 2 * giai_thua_de_quy(1)
# = 5 * 4 * 3 * 2 * 1 * 1     (n=1 → trả 1, dừng)
# = 120
⚠️ Quên điều kiện dừng = vô hạn

Nếu hàm đệ quy không có điều kiện dừng → gọi mãi mãi → sập chương trình (Stack Overflow).

# SAI — không có điều kiện dừng
def de_quy_vo_han(n):
    return n * de_quy_vo_han(n - 1)    # đệ quy mãi → sập

3. Bước 1: Chia đôi (Fast/Slow pointer)

Để chia DSLK thành 2 nửa, ta dùng kỹ thuật Fast/Slow pointer:

  • con_tro_cham đi 1 bước mỗi lần.
  • con_tro_nhanh đi 2 bước mỗi lần.
  • Khi con_tro_nhanh tới cuối → con_tro_cham đang ở giữa.
def _chia_doi(self):
    """
    Chia danh sách thành 2 nửa bằng Fast/Slow pointer.
    Trả về tuple (nua_dau, nua_sau) — head của 2 nửa.
    """
    con_tro_cham = self.head
    con_tro_nhanh = self.head

    # Lưu ý: kiểm tra next.next để không bị vượt quá cuối
    while con_tro_nhanh.next is not None and \
          con_tro_nhanh.next.next is not None:
        con_tro_cham = con_tro_cham.next          # đi 1 bước
        con_tro_nhanh = con_tro_nhanh.next.next   # đi 2 bước

    # Khi thoát vòng lặp, con_tro_cham đang ở giữa
    nua_dau = self.head
    nua_sau = con_tro_cham.next                    # nửa sau bắt đầu từ Node kế slow

    # Cắt liên kết giữa 2 nửa
    con_tro_cham.next = None
    if nua_sau is not None:
        nua_sau.truoc = None

    return (nua_dau, nua_sau)

Mô phỏng chia đôi

DSLK: A ⇄ B ⇄ C ⇄ D ⇄ E

Vòng lặp 1
A (slow) ⇄ B ⇄ C ⇄ D ⇄ E (fast)

slow ở A, fast ở E → fast.next = None → thoát vòng

Nhưng thực tế fast.next.next = None (vì fast ở E, next=None) → thoát sớm hơn

Để đơn giản, hãy tưởng tượng DSLK dài hơn: A ⇄ B ⇄ C ⇄ D ⇄ E ⇄ F ⇄ G

Bước đầu: slow=A, fast=A
A (slow, fast) ⇄ B ⇄ C ⇄ D ⇄ E ⇄ F ⇄ G
Vòng 1: slow=B, fast=C
A ⇄ B (slow) ⇄ C (fast) ⇄ D ⇄ E ⇄ F ⇄ G
Vòng 2: slow=C, fast=E
A ⇄ B ⇄ C (slow) ⇄ D ⇄ E (fast) ⇄ F ⇄ G
Vòng 3: slow=D, fast=G → fast.next=None → DỪNG
A ⇄ B ⇄ C ⇄ D (slow) ⇄ E ⇄ F ⇄ G (fast)

slow đang ở D → chốt! Nửa đầu: A→D, nửa sau: E→G

4. Bước 2: Trộn 2 danh sách đã sort

Sau khi có 2 nửa đã sắp xếp, ta trộn (merge) chúng thành 1 danh sách cũng sắp xếp.

Ý tưởng: so sánh Node đầu của A với Node đầu của B, lấy Node nhỏ hơn gắn vào kết quả.

def _tron(self, ds_a, ds_b, tieu_chi):
    """
    Trộn 2 danh sách A và B (đã sắp xếp) thành 1 danh sách cũng sắp xếp.
    """
    # Node tạm (Node(None)) làm điểm xuất phát để dễ code
    node_tam = Node(None)
    hien_tai = node_tam                       # con trỏ xây danh sách kết quả
    node_a = ds_a.head
    node_b = ds_b.head

    # So sánh từng cặp Node đầu của A và B
    while node_a is not None and node_b is not None:
        gia_tri_a = self._lay_gia_tri(node_a.data, tieu_chi)
        gia_tri_b = self._lay_gia_tri(node_b.data, tieu_chi)

        # Lấy Node có giá trị nhỏ hơn gắn vào kết quả
        if gia_tri_a <= gia_tri_b:
            hien_tai.next = node_a
            node_a.truoc = hien_tai
            node_a = node_a.next              # tiến A
        else:
            hien_tai.next = node_b
            node_b.truoc = hien_tai
            node_b = node_b.next              # tiến B
        hien_tai = hien_tai.next

    # Một trong 2 danh sách còn sót → gắn toàn bộ vào cuối kết quả
    if node_a is not None:
        hien_tai.next = node_a
        node_a.truoc = hien_tai
        self.cuoi = ds_a.cuoi
    if node_b is not None:
        hien_tai.next = node_b
        node_b.truoc = hien_tai
        self.cuoi = ds_b.cuoi

    # Bỏ Node tạm và đặt lại head mới
    self.head = node_tam.next
    if self.head is not None:
        self.head.truoc = None                # head mới không có truoc

def _lay_gia_tri(self, doi_tuong, tieu_chi):
    """Lấy giá trị để so sánh theo tiêu chí."""
    if tieu_chi == 'gia_ban':
        if hasattr(doi_tuong, 'tinh_gia_ban'):
            return doi_tuong.tinh_gia_ban()   # đa hình!
        return getattr(doi_tuong, 'gia_co_ban', 0)
    elif tieu_chi == 'ten_san_pham':
        return getattr(doi_tuong, 'ten_san_pham', '')
    elif tieu_chi == 'ton_kho':
        return getattr(doi_tuong, 'ton_kho', 0)
    return 0

Mô phỏng trộn

Trộn A = [2, 5, 8] với B = [1, 3, 9] thành [1, 2, 3, 5, 8, 9]:

Bước đầu: node_a=2, node_b=1
2→ 5→ 8 + 1→ 3→ 9

2 vs 1 → 1 nhỏ hơn → lấy 1. Kết quả: [1]

Bước 2: node_a=2, node_b=3
2→ 5→ 8 + 1→ 3→ 9

2 vs 3 → 2 nhỏ hơn → lấy 2. Kết quả: [1, 2]

Bước 3: node_a=5, node_b=3 → lấy 3. Kết quả: [1, 2, 3]

Bước 4: node_a=5, node_b=9 → lấy 5. Kết quả: [1, 2, 3, 5]

Bước 5: node_a=8, node_b=9 → lấy 8. Kết quả: [1, 2, 3, 5, 8]

Bước 6: A hết → gắn nguyên phần còn lại của B. Kết quả: [1, 2, 3, 5, 8, 9] ✓

5. Bước 3: Ghép 2 bước lại (hàm sap_xep_tron)

def sap_xep_tron(self, tieu_chi='gia_ban'):
    """
    Sắp xếp DSLK bằng Merge Sort — O(n log n).

    Hỗ trợ 3 tiêu chí: 'gia_ban', 'ten_san_pham', 'ton_kho'.
    """
    # Điều kiện dừng: 0 hoặc 1 phần tử → đã "sắp xếp" sẵn
    if self.head is None or self.head.next is None:
        return

    # 1. Chia đôi danh sách thành 2 nửa
    nua_dau, nua_sau = self._chia_doi()

    # 2. Bọc nửa đầu thành DoublyLinkedList để đệ quy
    ds_nua_dau = DoublyLinkedList()
    ds_nua_dau.head = nua_dau
    # Tìm Node cuối của nửa đầu (vì _chia_doi chỉ trả head)
    node_cuoi_dau = nua_dau
    while node_cuoi_dau.next is not None:
        node_cuoi_dau = node_cuoi_dau.next
    ds_nua_dau.cuoi = node_cuoi_dau
    ds_nua_dau.so_luong = self._dem_node(nua_dau)

    # Tương tự cho nửa sau
    ds_nua_sau = DoublyLinkedList()
    ds_nua_sau.head = nua_sau
    node_cuoi_sau = nua_sau
    while node_cuoi_sau.next is not None:
        node_cuoi_sau = node_cuoi_sau.next
    ds_nua_sau.cuoi = node_cuoi_sau
    ds_nua_sau.so_luong = self._dem_node(nua_sau)

    # Đệ quy sắp xếp từng nửa
    ds_nua_dau.sap_xep_tron(tieu_chi)         # GỌI LẠI CHÍNH NÓ!
    ds_nua_sau.sap_xep_tron(tieu_chi)

    # 3. Trộn 2 nửa đã sắp xếp vào lại self
    self._tron(ds_nua_dau, ds_nua_sau, tieu_chi)

def _dem_node(self, node_dau):
    """Đếm số Node từ node_dau đến cuối."""
    dem = 0
    node_hien_tai = node_dau
    while node_hien_tai is not None:
        dem += 1
        node_hien_tai = node_hien_tai.next
    return dem

6. Ví dụ đầy đủ

Sort DSLK [38, 27, 43, 3, 9, 82, 10] theo giá trị tăng dần:

# Tạo DSLK với 7 số
ds = DoublyLinkedList()
for x in [38, 27, 43, 3, 9, 82, 10]:
    ds.them_vao_cuoi(x)

# Sort
ds.sap_xep_tron()

# In kết quả
print(ds)
# 3 <-> 9 <-> 10 <-> 27 <-> 38 <-> 43 <-> 82

Đường đi của đệ quy (mỗi dòng = 1 lần gọi sap_xep_tron):

sap_xep_tron([38, 27, 43, 3, 9, 82, 10])
├── _chia_doi → [38, 27, 43, 3] + [9, 82, 10]
│
├── sap_xep_tron([38, 27, 43, 3])          ← đệ quy nửa đầu
│   ├── _chia_doi → [38, 27] + [43, 3]
│   ├── sap_xep_tron([38, 27])
│   │   ├── _chia_doi → [38] + [27]
│   │   ├── sap_xep_tron([38])    → return (1 ptu, dừng)
│   │   ├── sap_xep_tron([27])    → return (1 ptu, dừng)
│   │   └── _tron([38], [27])     → [27, 38]
│   ├── sap_xep_tron([43, 3])
│   │   ├── _chia_doi → [43] + [3]
│   │   ├── sap_xep_tron([43])    → return
│   │   ├── sap_xep_tron([3])     → return
│   │   └── _tron([43], [3])      → [3, 43]
│   └── _tron([27, 38], [3, 43])  → [3, 27, 38, 43]
│
├── sap_xep_tron([9, 82, 10])              ← đệ quy nửa sau
│   ├── _chia_doi → [9, 82] + [10]
│   ├── sap_xep_tron([9, 82])
│   │   ├── _chia_doi → [9] + [82]
│   │   ├── _tron([9], [82])       → [9, 82]
│   ├── sap_xep_tron([10])         → return
│   └── _tron([9, 82], [10])       → [9, 10, 82]
│
└── _tron([3, 27, 38, 43], [9, 10, 82])  → [3, 9, 10, 27, 38, 43, 82] ✓

7. Độ phức tạp O(n log n)

Merge Sort chia đôi liên tục → số lần chia = log₂(n).

Mỗi lần trộn cần O(n) thao tác.

Tổng: n × log₂(n) = O(n log n).

nBubble Sort O(n²)Merge Sort O(n log n)
1010033
10010,000664
1,0001,000,0009,965
10,000100,000,000132,877

Nếu sort 10,000 mặt hàng, Bubble Sort cần 100 triệu thao tác, Merge Sort chỉ cần ~133 nghìn. Nhanh hơn 750 lần!
