🚶 Bài 8: Queue FIFO (Hàng đợi)

Queue (Hàng đợi) là cấu trúc dữ liệu theo nguyên lý FIFO — First In First Out (Vào trước Ra trước). Phần tử nào được thêm vào hàng trước thì sẽ được lấy ra trước, giống như xếp hàng mua vé: người đến trước được phục vụ trước.

Trong đồ án, Queue được dùng cho Giỏ hàng + Thanh toán (CN09). Khi khách thêm sản phẩm A rồi mới thêm B, hệ thống thanh toán sẽ xử lý A trước rồi mới tới B.

1. Khái niệm Queue FIFO

Queue là cấu trúc dữ liệu tuyến tính có 2 đầu:

  • Đầu (dau): nơi phần tử được lấy ra (dequeue).
  • Cuối (cuoi_hang): nơi phần tử được thêm vào (enqueue).

Nguyên lý FIFO (First In First Out): phần tử vào trước sẽ ra trước.

📖 So sánh với Stack

Stack LIFO (bài 7): chỉ có 1 đầu (đỉnh). Cả push và pop đều ở đỉnh → phần tử vào sau sẽ ra trước.

Queue FIFO (bài này): có 2 đầu. Thêm ở cuối, lấy ở đầu → phần tử vào trước sẽ ra trước.

2. Minh họa trực quan

Queue — 4 món trong giỏ hàng

Thêm vào cuối ↓    |    Lấy ra đầu ↓

← Lấy ra A: Truyện Kiều B: Tạp chí C: Báo giấy D: Luận văn ← Thêm vào

A vào trước → khi thanh toán, A sẽ được xử lý đầu tiên, D cuối cùng.

Các thao tác cơ bản:

  • them_vao(data) (Enqueue): thêm phần tử vào cuối hàng.
  • lay_ra() (Dequeue): lấy phần tử ở đầu hàng ra và xóa.
  • xem_danh_sach(): duyệt từ đầu đến cuối, không xóa.
  • rong(): kiểm tra hàng có rỗng không.
  • xoa_tat_ca(): xóa toàn bộ hàng.

3. Code hoàn chỉnh (file cau_truc_du_lieu.py)

Đây là phần code tự cài đặt Queue trong file loi_thuat_toan/cau_truc_du_lieu.py — không dùng list Python, mà tự tạo Node liên kết qua con trỏ.

class NodeHangDoi:
    """
    Lớp NodeHangDoi — một mắt xích trong Hàng đợi.

    Mỗi Node chứa:
        data: Dữ liệu (ở đây là mặt hàng trong giỏ).
        sau:  Con trỏ trỏ đến Node phía sau (Node tiếp theo trong hàng).
    """

    def __init__(self, data):
        """Khởi tạo Node mới với dữ liệu truyền vào, sau = None."""
        self.data = data        # Lưu mặt hàng (đối tượng Sach/TapChi/...).
        self.sau = None         # Chưa trỏ đến Node kế tiếp.


class HangDoi:
    """
    Lớp HangDoi (Queue) — tự cài đặt theo nguyên lý FIFO.

    Thuộc tính:
        dau:       Trỏ đến Node đầu hàng (lấy ra ở đây khi thanh toán).
        cuoi_hang: Trỏ đến Node cuối hàng (thêm mới vào đây).
        so_luong:  Số Node hiện có.
    """

    def __init__(self):
        """Khởi tạo hàng đợi rỗng: đầu = cuối = None, số lượng = 0."""
        self.dau = None
        self.cuoi_hang = None
        self.so_luong = 0

    def them_vao(self, data):
        """Enqueue — thêm một phần tử vào CUỐI hàng đợi."""
        node_moi = NodeHangDoi(data)        # Bọc dữ liệu vào Node mới.

        if self.dau is None:                # Trường hợp hàng đang rỗng.
            self.dau = node_moi             # Node mới là cả đầu lẫn cuối.
            self.cuoi_hang = node_moi
        else:                               # Trường hợp đã có phần tử.
            self.cuoi_hang.sau = node_moi   # Nối Node mới sau Node cuối.
            self.cuoi_hang = node_moi       # Cập nhật cuối = Node mới.

        self.so_luong += 1

    def lay_ra(self):
        """Dequeue — lấy phần tử ở ĐẦU hàng đợi ra (và xóa nó)."""
        if self.dau is None:
            return None

        data = self.dau.data                # Lưu dữ liệu đầu.
        self.dau = self.dau.sau             # Dịch đầu sang Node kế tiếp.

        if self.dau is None:                # Nếu sau khi dịch mà rỗng →
            self.cuoi_hang = None           # phải reset cả cuối_hang.

        self.so_luong -= 1
        return data

    def xem_danh_sach(self):
        """Trả về list Python chứa toàn bộ phần tử từ đầu đến cuối."""
        ket_qua = []
        node_hien_tai = self.dau
        while node_hien_tai is not None:
            if hasattr(node_hien_tai.data, 'chuyen_thanh_dict'):
                ket_qua.append(node_hien_tai.data.chuyen_thanh_dict())
            else:
                ket_qua.append(node_hien_tai.data)
            node_hien_tai = node_hien_tai.sau
        return ket_qua

    def rong(self):
        """Kiểm tra hàng đợi có rỗng không."""
        return self.dau is None

    def xoa_tat_ca(self):
        """Clear — xóa toàn bộ giỏ hàng."""
        self.dau = None
        self.cuoi_hang = None
        self.so_luong = 0

    def xoa_theo_ma(self, ma_so):
        """Xóa một phần tử BẤT KỲ trong hàng đợi theo mã số."""
        if self.dau is None:
            return False

        def _lay_ma(d):
            if hasattr(d, 'ma_so'):
                return d.ma_so
            if isinstance(d, dict):
                return d.get('ma_so')
            return None

        # Trường hợp 1: Node cần xóa chính là Node đầu.
        if _lay_ma(self.dau.data) == ma_so:
            self.dau = self.dau.sau
            if self.dau is None:
                self.cuoi_hang = None
            self.so_luong -= 1
            return True

        # Trường hợp 2: Node cần xóa ở giữa/cuối → duyệt tìm.
        node_hien_tai = self.dau
        while node_hien_tai.sau is not None:
            if _lay_ma(node_hien_tai.sau.data) == ma_so:
                if node_hien_tai.sau == self.cuoi_hang:
                    self.cuoi_hang = node_hien_tai
                node_hien_tai.sau = node_hien_tai.sau.sau
                self.so_luong -= 1
                return True
            node_hien_tai = node_hien_tai.sau

        return False

4. Lớp NodeHangDoi

Mỗi phần tử trong hàng được bọc trong 1 Node. Node chứa:

  • data: dữ liệu thực (món hàng trong giỏ).
  • sau: con trỏ trỏ tới Node kế tiếp (phía sau).
class NodeHangDoi:
    def __init__(self, data):
        self.data = data
        self.sau = None    # Ban đầu chưa trỏ tới ai
💡 Tại sao gọi là "sau"?

Vì Node này trỏ tới Node phía sau nó trong hàng. Nếu bạn đang ở đầu hàng (vị trí 1), .sau trỏ tới vị trí 2.

Khác với Stack (chỉ có duoi), Queue dùng sau vì có thể duyệt tiến từ đầu đến cuối.

5. Hàm them_vao (Enqueue)

Thêm 1 phần tử vào cuối hàng. Có 2 trường hợp:

Trường hợp 1: Hàng rỗng (chưa có Node nào)

if self.dau is None:
    self.dau = node_moi             # Node mới là đầu
    self.cuoi_hang = node_moi       # Và cũng là cuối

Trường hợp 2: Hàng đã có phần tử

else:
    self.cuoi_hang.sau = node_moi   # Nối Node mới sau Node cuối
    self.cuoi_hang = node_moi       # Cập nhật cuối = Node mới
Trực quan them_vao khi hàng đã có dữ liệu

Trước khi thêm D:

A → B → C

Sau khi thêm D:

A → B → C → D (mới)

cuoi_hang dịch từ C sang D.

Độ phức tạp: O(1) vì đã giữ sẵn con trỏ cuoi_hang. Nếu chỉ có dau thì phải duyệt O(n) để tìm cuối.

6. Hàm lay_ra (Dequeue)

Lấy phần tử ở đầu hàng ra và xóa nó khỏi hàng.

def lay_ra(self):
    if self.dau is None:
        return None              # Hàng rỗng → không có gì để lấy

    data = self.dau.data         # Lưu dữ liệu đầu trước khi xóa
    self.dau = self.dau.sau      # Dịch đầu sang Node kế tiếp

    if self.dau is None:         # Nếu sau khi dịch mà rỗng →
        self.cuoi_hang = None    # phải reset cả cuối_hang (tránh tham chiếu rác)

    self.so_luong -= 1
    return data
⚠️ Tại sao phải reset cuoi_hang khi rỗng?

Vì nếu hàng chỉ còn 1 Node, khi dequeue thì dau sẽ thành None, nhưng cuoi_hang vẫn trỏ tới Node đã bị gỡ — đó là tham chiếu rác.

Để tránh bug, ta reset cuoi_hang = None khi dau thành None.

Trực quan lay_ra

Trước khi lay_ra:

A (đầu) → B → C

Sau khi lay_ra — A bị lấy ra, B thành đầu:

B (đầu mới) → C

Hàm trả về dữ liệu của A (đã bị xóa khỏi hàng).

7. Hàm xem_danh_sach

Trả về list Python chứa toàn bộ phần tử từ đầu đến cuối — không xóa gì cả. Dùng để UI hiển thị giỏ hàng.

def xem_danh_sach(self):
    ket_qua = []
    node_hien_tai = self.dau                # Bắt đầu duyệt từ đầu
    while node_hien_tai is not None:        # Lặp đến khi hết Node
        if hasattr(node_hien_tai.data, 'chuyen_thanh_dict'):
            ket_qua.append(node_hien_tai.data.chuyen_thanh_dict())
        else:
            ket_qua.append(node_hien_tai.data)
        node_hien_tai = node_hien_tai.sau   # Tiến sang Node kế
    return ket_qua
📖 Tại sao dùng hasattr?

Vì data trong giỏ có thể là:

  • Đối tượng OOP (Sach, TapChi, ...) → có hàm chuyen_thanh_dict().
  • Dict Python (món hàng trong giỏ) → không có hàm này, chỉ cần giữ nguyên.

Dùng hasattr để kiểm tra trước khi gọi — tránh lỗi AttributeError.

8. Hàm xoa_theo_ma

Xóa 1 phần tử bất kỳ trong hàng theo mã số. Khách có thể đổi ý, muốn bỏ 1 món khỏi giỏ — không nhất thiết phải là món đầu.

def xoa_theo_ma(self, ma_so):
    if self.dau is None:
        return False

    def _lay_ma(d):
        if hasattr(d, 'ma_so'):
            return d.ma_so
        if isinstance(d, dict):
            return d.get('ma_so')
        return None

    # Trường hợp 1: Node cần xóa là Node đầu
    if _lay_ma(self.dau.data) == ma_so:
        self.dau = self.dau.sau
        if self.dau is None:
            self.cuoi_hang = None
        self.so_luong -= 1
        return True

    # Trường hợp 2: Node cần xóa ở giữa/cuối → duyệt tìm
    node_hien_tai = self.dau
    while node_hien_tai.sau is not None:
        if _lay_ma(node_hien_tai.sau.data) == ma_so:
            if node_hien_tai.sau == self.cuoi_hang:
                self.cuoi_hang = node_hien_tai
            node_hien_tai.sau = node_hien_tai.sau.sau
            self.so_luong -= 1
            return True
        node_hien_tai = node_hien_tai.sau

    return False

Thuật toán duyệt tìm

Khi Node cần xóa không phải đầu, ta duyệt từ đầu. Tại mỗi Node, kiểm tra Node kế tiếp (.sau) có phải Node cần xóa không. Nếu có, bỏ qua Node đó bằng cách nối .sau của Node hiện tại sang .sau.sau (Node sau Node cần xóa).

# Trước: A → B → C → D, muốn xóa C
node_hien_tai = B
node_hien_tai.sau = C   # cần xóa
node_hien_tai.sau.sau = D   # Node sau C

# Lệnh xóa:
node_hien_tai.sau = node_hien_tai.sau.sau   # B.sau = D

# Kết quả: A → B → D (C bị cắt)

9. Ứng dụng trong đồ án

Queue FIFO được dùng trong CN09 — Giỏ hàng + Thanh toán:

➕ Thêm vào giỏ
Khách chọn sản phẩm → them_vao (enqueue vào cuối).
👁️ Xem giỏ
Hiển thị toàn bộ giỏ → xem_danh_sach (không lấy ra).
❌ Bỏ món
Khách đổi ý → xoa_theo_ma.
💰 Thanh toán
Lần lượt lay_ra từng món (FIFO) → tính tổng tiền → trả hóa đơn.
✅ Tóm tắt
  • Queue FIFO: vào trước — ra trước (như xếp hàng mua vé).
  • 2 con trỏ chính: dau (lấy ra) + cuoi_hang (thêm vào).
  • Enqueue/Dequeue đều O(1).
  • Trong đồ án: dùng cho Giỏ hàng + Thanh toán (CN09).
