🔗 Bài 5: Danh sách liên kết đôi (Doubly Linked List)

Danh sách liên kết đôi (DSLK đôi) là cấu trúc dữ liệu tự cài đặt — không dùng list Python. Đây là phần Huy phụ trách.

🚨 Đây là phần KHÓ NHẤT đồ án

Nếu bạn chưa từng nghe "con trỏ", "node", "linked list" — đọc kỹ bài này. Huy phụ trách vì đây là phần khó nhất, không ai trong nhóm đã học.

1. DSLK là gì? Tại sao không dùng list Python?

List Python là dãy các phần tử liền nhau trong bộ nhớ:

List Python — phần tử liền nhau
[0] 10 [1] 20 [2] 30 [3] 40

Truy cập ds[2] → ngay ra 30 (O(1)). Nhưng chèn/xóa giữa danh sách → phải dời tất cả phần tử phía sau (O(n)).

DSLK — các phần tử (Node) rải rác trong bộ nhớ, liên kết qua con trỏ:

DSLK đôi — Node rải rác, liên kết qua con trỏ
10
⬅️ ➡️ ⇄ 20
⬅️ ➡️ ⇄ 30
⬅️ ➡️ ⇄ 40
⬅️ ➡️

Mỗi Node biết Node trước + Node sau. Chèn/xóa giữa → chỉ cần sửa vài con trỏ (O(1) nếu có Node tham chiếu).

Tại sao đồ án dùng DSLK?

Vì đề bài yêu cầu "tự cài đặt cấu trúc dữ liệu, không dùng built-in" — để sinh viên hiểu sâu về con trỏ + bộ nhớ. Đây là yêu cầu cốt lõi của môn Lập trình Nâng cao.

DSLK ĐÔI khác DSLK ĐƠN ở đâu?

  • DSLK đơn: mỗi Node chỉ có 1 con trỏ next → chỉ duyệt được 1 chiều (tiến).
  • DSLK đôi: mỗi Node có 2 con trỏ truoc + next → duyệt được 2 chiều (tiến + lùi).

2. Node — mắt xích của DSLK

Mỗi Node chứa 3 thứ: dữ liệu + con trỏ trước + con trỏ sau.

class Node:
    """
    Lớp Node — đơn vị cơ bản của DSLK đôi.

    Mỗi Node chứa:
        data  : Dữ liệu (thường là đối tượng MatHang).
        truoc : Con trỏ tới Node phía trước.
        next  : Con trỏ tới Node phía sau.
    """

    def __init__(self, data):
        self.data = data        # Dữ liệu (đối tượng sản phẩm)
        self.truoc = None       # Ban đầu chưa liên kết Node trước
        self.next = None        # Ban đầu chưa liên kết Node sau

    def __str__(self):
        return str(self.data)   # In Node → in dữ liệu bên trong
ℹ️ "Con trỏ" trong Python

Python không có con trỏ kiểu C. Thay vào đó, biến Python là tham chiếu (reference) tới đối tượng. Khi gán node.truoc = node_khac, node.truoc "trỏ tới" đối tượng node_khac.

None = "không trỏ tới đâu cả" (null).

3. Cấu trúc DSLK đôi

Lớp DoublyLinkedList quản lý toàn bộ danh sách. Chỉ cần giữ head (đầu) + cuoi (cuối) là duyệt được hết.

class DoublyLinkedList:
    """
    Lớp quản lý DSLK đôi.

    Thuộc tính:
        head      : Trỏ tới Node đầu (None nếu rỗng).
        cuoi      : Trỏ tới Node cuối (None nếu rỗng).
        so_luong  : Số Node hiện có (để tránh duyệt mỗi lần đếm).
    """

    def __init__(self):
        self.head = None        # Chưa có Node đầu
        self.cuoi = None        # Chưa có Node cuối
        self.so_luong = 0       # Đếm số phần tử

Minh họa DSLK rỗng vs DSLK có 3 Node:

DSLK rỗng

head = None, cuoi = None, so_luong = 0

DSLK có 3 Node: A → B → C
Node A
data="Sách 1" ⇄ Node B
data="Sách 2" ⇄ Node C
data="Sách 3"

head → A, cuoi → C, so_luong = 3
A.truoc = None (không có Node trước A)
A.next = B
B.truoc = A, B.next = C
C.truoc = B, C.next = None

4. them_vao_cuoi — thêm vào cuối

def them_vao_cuoi(self, data):
    """
    Thêm phần tử mới vào CUỐI danh sách.
    Độ phức tạp: O(1) nhờ duy trì sẵn con trỏ `cuoi`.
    """
    # Bọc dữ liệu vào Node mới
    node_moi = Node(data)

    if self.head is None:
        # Trường hợp 1: danh sách đang RỖNG
        # Node mới là cả đầu lẫn cuối
        self.head = node_moi
        self.cuoi = node_moi
    else:
        # Trường hợp 2: đã có phần tử
        # Nối Node mới SAU Node cuối hiện tại
        self.cuoi.next = node_moi        # Node cuối trỏ next tới Node mới
        node_moi.truoc = self.cuoi       # Node mới trỏ truoc ngược lại
        self.cuoi = node_moi             # Cập nhật cuoi = Node mới

    self.so_luong += 1                   # Tăng bộ đếm
🎬 Mô phỏng: them_vao_cuoi("Sách A") rồi them_vao_cuoi("Sách B")

Mô phỏng từng bước

Giả sử DSLK đang có 2 Node: A ⇄ B. Thêm Node C vào cuối:

Bước 1: Tạo node_moi = Node(C)
A ⇄ B + C (mới)
truoc=None, next=None
Bước 2: self.cuoi.next = node_moi (B.next = C)
A ⇄ B
next → C → C
Bước 3: node_moi.truoc = self.cuoi (C.truoc = B)
A ⇄ B ⇄ C
Bước 4: self.cuoi = node_moi (cuoi = C)
A ⇄ B ⇄ C ← cuoi

so_luong tăng từ 2 lên 3.

5. xoa_node — xóa phần tử

Xóa có 3 trường hợp: xóa Node đầu, xóa Node cuối, xóa Node giữa.

def xoa_node(self, ma_so):
    """
    Xóa Node đầu tiên có data.ma_so == ma_so.
    Trả về: True nếu xóa được, False nếu không tìm thấy.
    """
    node_hien_tai = self.head                       # Bắt đầu từ đầu

    while node_hien_tai is not None:                # Lặp đến hết
        # Kiểm tra Node hiện tại có ma_so khớp không
        if hasattr(node_hien_tai.data, 'ma_so') and \
           node_hien_tai.data.ma_so == ma_so:

            # Trường hợp 1: Node cần xóa là Node ĐẦU
            if node_hien_tai == self.head:
                self.head = node_hien_tai.next      # Đẩy head sang Node kế
                if self.head is not None:
                    self.head.truoc = None          # Node đầu mới không có truoc
                else:
                    self.cuoi = None                # Nếu rỗng → reset cả cuoi

            # Trường hợp 2: Node cần xóa là Node CUỐI
            elif node_hien_tai == self.cuoi:
                self.cuoi = node_hien_tai.truoc     # Lùi cuoi về Node trước
                self.cuoi.next = None               # Cắt liên kết tới Node bị xóa

            # Trường hợp 3: Node ở GIỮA → nối prev và next bỏ qua Node hiện tại
            else:
                node_hien_tai.truoc.next = node_hien_tai.next
                node_hien_tai.next.truoc = node_hien_tai.truoc

            self.so_luong -= 1
            return True                              # Đã xóa

        node_hien_tai = node_hien_tai.next          # Tiến sang Node kế

    return False                                    # Duyệt hết mà không thấy

Mô phỏng xóa Node giữa

DSLK: A ⇄ B ⇄ C ⇄ D. Xóa Node B:

Trước khi xóa
A ⇄ B (xóa) ⇄ C ⇄ D
Bước 1: B.truoc.next = B.next (A.next = C)
A
next → C → B ⇄ C ⇄ D
Bước 2: B.next.truoc = B.truoc (C.truoc = A)
A ⇄ B ⇄ C
truoc → A ⇄ D
Sau khi xóa: A ⇄ C ⇄ D
A ⇄ C ⇄ D

Node B bị cắt khỏi danh sách. Python GC sẽ tự dọn rác.

6. Tìm kiếm (3 cách)

# Cách 1: tìm chính xác theo mã số → trả về 1 đối tượng
def tim_theo_ma_so(self, ma_so):
    """Tìm chính xác theo mã. O(n). Trả về đối tượng hoặc None."""
    node_hien_tai = self.head
    while node_hien_tai is not None:
        if hasattr(node_hien_tai.data, 'ma_so') and \
           node_hien_tai.data.ma_so == ma_so:
            return node_hien_tai.data           # trả về data, không phải Node
        node_hien_tai = node_hien_tai.next
    return None                                 # không tìm thấy

# Cách 2: tìm theo tên (so khớp CHỨA, không phân biệt HOA/thường)
def tim_kiem(self, tu_khoa):
    """Trả về list các đối tượng có tên chứa tu_khoa."""
    ket_qua = []
    node_hien_tai = self.head
    tu_khoa_chu_thuong = tu_khoa.lower()        # chuẩn hóa
    while node_hien_tai is not None:
        if hasattr(node_hien_tai.data, 'ten_san_pham'):
            ten_chu_thuong = node_hien_tai.data.ten_san_pham.lower()
            # in → kiểm tra "chứa"
            if tu_khoa_chu_thuong in ten_chu_thuong:
                ket_qua.append(node_hien_tai.data)
        node_hien_tai = node_hien_tai.next
    return ket_qua

# Cách 3: tìm theo loại hàng
def tim_theo_the_loai(self, the_loai):
    """Trả về list các đối tượng có loại khớp."""
    ket_qua = []
    node_hien_tai = self.head
    the_loai_chu_thuong = the_loai.lower()
    while node_hien_tai is not None:
        if hasattr(node_hien_tai.data, 'loai_hang'):
            if the_loai_chu_thuong in node_hien_tai.data.loai_hang.lower():
                ket_qua.append(node_hien_tai.data)
        node_hien_tai = node_hien_tai.next
    return ket_qua

Tìm kiếm nâng cao (theo cả mã + tên + loại)

def tim_kiem_nang_cao(self, tu_khoa):
    """Tìm đồng thời theo MÃ + TÊN + LOẠI. Dùng set để tránh trùng."""
    ket_qua = []
    ma_da_them = set()                          # set không chứa phần tử trùng
    node_hien_tai = self.head
    tu_khoa_lower = tu_khoa.lower()

    while node_hien_tai is not None:
        data = node_hien_tai.data
        da_khop = False

        # Kiểm tra khớp theo mã số
        if hasattr(data, 'ma_so') and tu_khoa_lower in data.ma_so.lower():
            da_khop = True

        # Nếu chưa khớp → kiểm tra theo tên
        if not da_khop and hasattr(data, 'ten_san_pham') and \
           tu_khoa_lower in data.ten_san_pham.lower():
            da_khop = True

        # Nếu vẫn chưa khớp → kiểm tra theo loại
        if not da_khop and hasattr(data, 'loai_hang') and \
           tu_khoa_lower in data.loai_hang.lower():
            da_khop = True

        # Nếu khớp và chưa thêm → thêm vào kết quả
        if da_khop and data.ma_so not in ma_da_them:
            ket_qua.append(data)
            ma_da_them.add(data.ma_so)         # đánh dấu đã thêm

        node_hien_tai = node_hien_tai.next

    return ket_qua
ℹ️ Tại sao dùng set?

Nếu sản phẩm có mã "S001" và tên "S001 - Sách", tìm "S001" sẽ khớp cả 2 điều kiện → nếu không dùng set, sản phẩm bị thêm 2 lần.

Set kiểm tra in nhanh hơn list (O(1) vs O(n)).

7. cap_nhat_node — sửa phần tử

def cap_nhat_node(self, ma_so, du_lieu_moi):
    """
    Cập nhật các trường của Node có ma_so.
    du_lieu_moi là dict {ten_truong: gia_tri_moi}.
    Trả về đối tượng đã cập nhật, hoặc None nếu không tìm thấy.
    """
    node_hien_tai = self.head
    while node_hien_tai is not None:
        if hasattr(node_hien_tai.data, 'ma_so') and \
           node_hien_tai.data.ma_so == ma_so:
            doi_tuong = node_hien_tai.data

            # Lặp từng cặp (tên_trường, giá_trị) trong dict
            for thuoc_tinh, gia_tri in du_lieu_moi.items():
                if hasattr(doi_tuong, thuoc_tinh):
                    # setattr(obj, "tên", giá_trị) = obj.tên = giá_trị
                    setattr(doi_tuong, thuoc_tinh, gia_tri)
            return doi_tuong
        node_hien_tai = node_hien_tai.next
    return None

Ví dụ sử dụng:

# Tạo đối tượng + DSLK
sach = Sach("MH001", "Truyện Kiều", 50000, 20, "Nguyễn Du", "NXB Văn Học")
dslk = DoublyLinkedList()
dslk.them_vao_cuoi(sach)

# Sửa giá + tồn kho
dslk.cap_nhat_node("MH001", {"gia_co_ban": 55000, "ton_kho": 25})

# Tương đương:
# sach.gia_co_ban = 55000
# sach.ton_kho = 25

8. Duyệt + chuyen_thanh_danh_sach

def chuyen_thanh_danh_sach(self):
    """
    Đổi DSLK thành list Python các dict.
    Dùng khi trả về JSON cho frontend.
    """
    ket_qua = []
    node_hien_tai = self.head
    while node_hien_tai is not None:
        du_lieu = node_hien_tai.data
        if hasattr(du_lieu, 'chuyen_thanh_dict'):
            # Nếu là đối tượng OOP → gọi chuyen_thanh_dict()
            ket_qua.append(du_lieu.chuyen_thanh_dict())
        else:
            ket_qua.append(str(du_lieu))
        node_hien_tai = node_hien_tai.next
    return ket_qua

def __str__(self):
    """In DSLK theo dạng: A <-> B <-> C."""
    danh_sach = []
    node_hien_tai = self.head
    while node_hien_tai is not None:
        danh_sach.append(str(node_hien_tai.data))
        node_hien_tai = node_hien_tai.next
    return ' <-> '.join(danh_sach)

def __len__(self):
    """Cho phép dùng len(ds) — trả về so_luong."""
    return self.so_luong

Ví dụ sử dụng:

ds = DoublyLinkedList()
ds.them_vao_cuoi(Sach("MH001", "Truyện Kiều", 50000, 20, ...))
ds.them_vao_cuoi(TapChi("MH002", "Tạp chí KH", 30000, 15, ...))

print(len(ds))                    # 2 (gọi __len__)
print(ds)                         # [MH001] Sách - Truyện Kiều ... <-> [MH002] ...

# Chuyển sang list dict (để hiện trong Treeview)
list_dict = ds.chuyen_thanh_danh_sach()
# [{'ma_so': 'MH001', 'ten': 'Truyện Kiều', ...}, {'ma_so': 'MH002', ...}]
