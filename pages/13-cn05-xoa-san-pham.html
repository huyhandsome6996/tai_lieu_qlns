🗑️ CN05: Xóa sản phẩm

Chức năng Xóa cho phép admin xóa sản phẩm khỏi hệ thống. Nhân viên không được phép xóa — nút Xóa bị vô hiệu hóa trong UI và thuật toán cũng kiểm tra quyền.

🔒 Yêu cầu phân quyền

Chức năng Xóa yêu cầu vai trò admin. Nhân viên không thấy nút Xóa hoạt động (nút bị disabled) và nếu somehow gọi thuật toán, nó vẫn từ chối với thông báo rõ ràng.

2. File _tt: xoa_san_pham

from loi_thuat_toan.ket_noi_sqlite import xoa_mot_san_pham
from loi_thuat_toan.danh_sach_lien_ket import DoublyLinkedList


def xoa_san_pham(danh_sach, bo_undoredo, ma_so, la_admin):
    """
    Xóa 1 sản phẩm theo mã số.

    Tham số:
        danh_sach   : DSLK đôi toàn cục.
        bo_undoredo : dict 2 Stack undo/redo.
        ma_so       : mã sản phẩm cần xóa.
        la_admin    : True nếu user hiện tại là admin (được phép xóa).
    """
    try:
        # === BƯỚC 1: KIỂM TRA PHÂN QUYỀN ===
        if not la_admin:
            return False, 'BẠN LÀ NHÂN VIÊN - KHÔNG ĐƯỢC PHÉP XÓA SẢN PHẨM.'

        # === BƯỚC 2: TÌM SẢN PHẨM CẦN XÓA ===
        sp_cu = danh_sach.tim_theo_ma_so(ma_so)
        if sp_cu is None:
            return False, f'Không tìm thấy sản phẩm mã {ma_so}.'

        # === BƯỚC 3: LƯU DỮ LIỆU CŨ (để Undo có thể khôi phục lại) ===
        du_lieu_cu = sp_cu.chuyen_thanh_dict()

        # === BƯỚC 4: XÓA KHỎI DSLK + SQLITE ===
        if danh_sach.xoa_node(ma_so):
            xoa_mot_san_pham(ma_so)

            # === BƯỚC 5: GHI LỊCH SỬ UNDO ===
            bo_undoredo['undo'].day_vao({
                'loai': 'xoa',
                'du_lieu_cu': du_lieu_cu,
                'du_lieu_moi': None
            })
            # Xóa toàn bộ Redo vì có thao tác mới
            bo_undoredo['redo'].xoa_tat_ca()

            return True, f'Đã xóa sản phẩm mã {ma_so}.'

        return False, 'Không tìm thấy sản phẩm.'
    except Exception as loi:
        return False, f'Lỗi: {loi}'

3. File _ui: hoi_xac_nhan_xoa

🎨 Hình minh hoạ giao diện: Chức năng xóa không có form nhập liệu — chỉ hiện 1 hộp thoại YES/NO hỏi user có chắc muốn xóa không. Nếu user nhấn Yes mới thực sự xóa. Đây là 3 hình minh hoạ các loại messagebox mà xoa_san_pham_ui.py dùng.

⚠️ Xác nhận xóa

Hình: messagebox.showerror(...) — tiêu đề đỏ, icon dấu X. Dùng khi xóa thất bại (không tìm thấy mã).

Hình: messagebox.showinfo(...) — tiêu đề xanh, icon i. Dùng khi xóa thành công + lưu vào stack Undo.

📋 Vì sao cn05 không cần lớp UI?

  • Không form thao tác xóa không cần user nhập liệu gì thêm — chỉ cần câu hỏi YES/NO.
  • 3 hàm tiện ích thay vì 1 lớp: hoi_xac_nhan_xoa(), bao_loi(), bao_thanh_cong() — gọi thẳng, không cần lưu trạng thái.
  • askyesno trả về True/False — Điều khiển kiểm tra giá trị trả về để quyết định có gọi thuật toán xoa_san_pham hay không.
  • Lưu ý Undo dòng chữ "thao tác này có thể Hoàn tác" nhắc user biết: nếu lỡ xóa, vẫn có thể Undo để khôi phục.

File UI của cn05 không phải lớp mà là các hàm tiện ích — vì xóa không cần form phức tạp, chỉ cần hộp thoại xác nhận YES/NO.

from tkinter import messagebox


def hoi_xac_nhan_xoa(parent, ma_so):
    """Hiện hộp thoại YES/NO để xác nhận xóa."""
    return messagebox.askyesno(
        'Xác nhận xóa',
        f'Bạn có chắc muốn xóa sản phẩm mã "{ma_so}"?\n'
        f'(Có thể hoàn tác bằng nút Undo)',
        parent=parent
    )


def bao_loi(parent, thong_bao):
    """Hiện hộp thoại lỗi."""
    messagebox.showerror('Lỗi', thong_bao, parent=parent)


def bao_thanh_cong(parent, thong_bao):
    """Hiện hộp thoại thành công."""
    messagebox.showinfo('Thành công', thong_bao, parent=parent)

Cú pháp messagebox

messagebox là module của Tkinter cung cấp các hộp thoại tiêu chuẩn. Các hàm chính:

Hàm Trả về Hiển thị
showinfo(title, msg) None Thông báo thông tin (icon "i")
showwarning(title, msg) None Cảnh báo (icon "!")
showerror(title, msg) None Lỗi (icon "X" đỏ)
askyesno(title, msg) True / False Hỏi Yes/No (Yes=True, No=False)
askokcancel(title, msg) True / False Hỏi OK/Cancel
askretrycancel(title, msg) True / False Hỏi Retry/Cancel
askyesnocancel(title, msg) True/False/None Hỏi Yes/No/Cancel (Cancel=None)

Tham số parent=parent: chỉ định cửa sổ cha — hộp thoại sẽ hiện modal trên parent. Nếu không truyền, hộp thoại có thể hiện sau các cửa sổ khác.

4. File _dk: DieuKhienXoaSanPham

from .xoa_san_pham_tt import xoa_san_pham
from .xoa_san_pham_ui import hoi_xac_nhan_xoa, bao_loi, bao_thanh_cong


class DieuKhienXoaSanPham:
    def __init__(self, danh_sach, bo_undoredo):
        self.danh_sach = danh_sach
        self.bo_undoredo = bo_undoredo

    def xu_ly_xoa(self, parent, ma_so, la_admin):
        """Xóa 1 sản phẩm theo mã số."""
        # Trường hợp không có mã
        if not ma_so:
            bao_loi(parent, 'Vui lòng chọn 1 dòng trong bảng để xóa.')
            return False

        # Hiện hộp thoại xác nhận YES/NO
        if not hoi_xac_nhan_xoa(parent, ma_so):
            return False   # User nhấn No → không làm gì

        # Gọi thuật toán xóa
        ok, msg = xoa_san_pham(self.danh_sach, self.bo_undoredo, ma_so, la_admin)
        if ok:
            bao_thanh_cong(parent, msg)
            return True
        else:
            bao_loi(parent, msg)
            return False

5. Undo entry khi xóa (lưu toàn bộ dữ liệu cũ)

Undo entry của thao tác Xóa có dạng:

{
    'loai': 'xoa',
    'du_lieu_cu': {                       # Toàn bộ dữ liệu sản phẩm đã xóa
        'ma_so': 'MH001',
        'loai_hang': 'Sách',
        'ten_san_pham': 'Truyện Kiều',
        'gia_co_ban': 50000,
        'ton_kho': 20,
        'gia_ban': 55000,
        'tac_gia': 'Nguyễn Du',
        'nha_xuat_ban': 'NXB Văn học'
    },
    'du_lieu_moi': None                   # Xóa → không có dữ liệu mới
}

Tại sao phải lưu toàn bộ dữ liệu cũ?

Vì khi Undo thao tác Xóa, ta cần thêm lại sản phẩm vào DSLK + SQLite. Nếu chỉ lưu mã số, ta không biết tên/giá/tồn kho/tác giả... để tạo lại đối tượng.

Đó là lý do du_lieu_cu phải là dict đầy đủ. Khi Undo:

  1. Lấy du_lieu_cu từ entry.
  2. Gọi _tao_doi_tuong_tu_dict(du_lieu_cu) — hàm này tạo đối tượng OOP (Sach/TapChi/...) dựa trên loai_hang.
  3. Gọi danh_sach.them_vao_cuoi(sp) + luu_mot_san_pham(sp).
def _tao_doi_tuong_tu_dict(d):
    """Tái tạo đối tượng MatHang từ dict (dùng cho Undo thao tác xóa)."""
    loai = d.get('loai_hang')
    if loai == 'Sách':
        return Sach(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                    d['ton_kho'], d.get('tac_gia', ''), d.get('nha_xuat_ban', ''))
    elif loai == 'Tạp chí':
        return TapChi(d['ma_so'], d['ten_san_pham'], d['gia_co_ban'],
                      d['ton_kho'], d.get('so_phat_hanh', ''))
    # ... các loại khác tương tự ...
💡 So sánh Undo entry của 3 thao tác
Thao tác du_lieu_cu du_lieu_moi
them None dict sản phẩm mới
sua dict cũ dict mới
xoa dict cũ (đầy đủ) None

Quy tắc: luôn lưu đủ dữ liệu để đảo ngược thao tác.
