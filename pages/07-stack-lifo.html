📚 Bài 7: Stack LIFO (Ngăn xếp)

Stack (ngăn xếp) là cấu trúc dữ liệu theo nguyên lý LIFO (Last In First Out — Vào sau, ra trước). Đây là phần Ngọ phụ trách, dùng cho chức năng Undo/Redo.

1. Nguyên lý LIFO

LIFO = Last In First Out: phần tử đẩy vào sau cùng sẽ được lấy ra đầu tiên.

Ví dụ thực tế: xếp đĩa. Đĩa xếp sau cùng nằm trên cùng → lấy ra trước.

Stack — LIFO
C ← Đỉnh (top) — lấy ra đầu tiên
B
A ← Đáy — lấy ra cuối cùng

day_vao(C) → ... → lay_ra() trả về C

2. NodeNganXep

Giống DSLK, Stack cũng tự cài bằng linked list. Mỗi Node chứa data + con trỏ duoi (trỏ tới Node ngay phía dưới).

class NodeNganXep:
    """
    Một mắt xích trong Ngăn xếp.

    Mỗi Node chứa:
        data : Dữ liệu (ở đây là thông tin thao tác để undo/redo).
        duoi : Con trỏ trỏ đến Node ngay phía dưới.
    """

    def __init__(self, data):
        self.data = data        # lưu dữ liệu (thường là dict thao tác)
        self.duoi = None        # chưa trỏ đến Node nào

3. Lớp NganXep

class NganXep:
    """
    Stack (Ngăn xếp) — tự cài đặt theo nguyên lý LIFO.

    Thuộc tính:
        dinh     : Trỏ đến Node trên cùng (None nếu rỗng).
        so_luong : Số Node hiện có.
    """

    def __init__(self):
        self.dinh = None        # chưa có Node → đỉnh = None
        self.so_luong = 0

4. day_vao — Push

Push = đẩy phần tử mới lên đỉnh.

def day_vao(self, data):
    """Push — đẩy phần tử mới lên ĐỈNH ngăn xếp."""
    # Bọc dữ liệu vào Node mới
    node_moi = NodeNganXep(data)

    # Node mới trỏ xuống Node đỉnh cũ
    node_moi.duoi = self.dinh

    # Cập nhật đỉnh = Node mới
    self.dinh = node_moi

    # Tăng bộ đếm
    self.so_luong += 1

Mô phỏng

Stack rỗng: dinh=None

(không có gì)

day_vao("A"): tạo node mới, dinh=A
A

A.duoi = None

day_vao("B"): B trỏ xuống A, dinh=B
B
A

B.duoi = A

day_vao("C"): C trỏ xuống B, dinh=C
C
B
A

C.duoi = B, B.duoi = A, A.duoi = None

5. lay_ra — Pop

Pop = lấy phần tử ở đỉnh ra (và xóa nó).

def lay_ra(self):
    """Pop — lấy phần tử trên ĐỈNH ra (và xóa nó)."""
    # Nếu stack rỗng → trả None
    if self.dinh is None:
        return None

    # Lưu dữ liệu đỉnh trước khi gỡ
    data = self.dinh.data

    # Hạ đỉnh xuống Node phía dưới
    self.dinh = self.dinh.duoi

    # Giảm bộ đếm
    self.so_luong -= 1

    # Trả về dữ liệu đã gỡ
    return data

Mô phỏng

Stack: C → B → A. lay_ra()
C ← bị lấy ra
B
A

data = "C", dinh = B

Sau khi lay_ra(): B → A
B ← đỉnh mới
A

Trả về "C"

6. xem_dinh — Peek

Peek = xem đỉnh mà không lấy ra.

def xem_dinh(self):
    """Peek — xem dữ liệu đỉnh mà không lấy ra."""
    if self.dinh is None:
        return None
    return self.dinh.data

7. rong + xoa_tat_ca

def rong(self):
    """Kiểm tra stack có rỗng không."""
    return self.dinh is None

def xoa_tat_ca(self):
    """Clear — xóa toàn bộ Node (chỉ cần cắt đỉnh)."""
    self.dinh = None                # cắt liên kết → Python GC dọn Node
    self.so_luong = 0

8. Ví dụ đầy đủ

from loi_thuat_toan.cau_truc_du_lieu import NganXep

stack = NganXep()

# Push 3 phần tử
stack.day_vao("Thao tac 1: them Sách A")
stack.day_vao("Thao tac 2: sua Tạp chí B")
stack.day_vao("Thao tac 3: xoa Báo giấy C")

print(f"Số phần tử: {stack.so_luong}")           # 3
print(f"Đỉnh (peek): {stack.xem_dinh()}")         # Thao tac 3: xoa Báo giấy C
print(f"Rỗng? {stack.rong()}")                    # False

# Pop — lấy ra theo LIFO
print(stack.lay_ra())                              # Thao tac 3: xoa Báo giấy C
print(stack.lay_ra())                              # Thao tac 2: sua Tạp chí B
print(stack.lay_ra())                              # Thao tac 1: them Sách A
print(stack.lay_ra())                              # None (stack rỗng)
print(f"Rỗng? {stack.rong()}")                    # True

Ứng dụng trong đồ án

Stack dùng cho Undo/Redo ở chức năng cn08_hoan_tac_lam_lai:

  • Mỗi lần user Thêm/Sửa/Xóa → day_vao() snapshot vào stack_undo.
  • User bấm Undo → lay_ra() khỏi stack_undo → đảo ngược thao tác → day_vao() vào stack_redo.
  • User bấm Redo → lay_ra() khỏi stack_redo → làm lại → day_vao() vào stack_undo.
ℹ️ Vì sao 2 Stack?

stack_undo lưu các thao tác ĐÃ LÀM (để hoàn tác).

stack_redo lưu các thao tác ĐÃ UNDO (để làm lại).

Khi user làm thao tác mới → stack_redo phải xoa_tat_ca() (vì không còn hợp lệ).
