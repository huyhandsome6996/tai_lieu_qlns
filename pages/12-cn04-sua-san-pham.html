✏️ CN04: Sửa sản phẩm

Chức năng Sửa cho phép cập nhật thông tin sản phẩm đã có. User chọn 1 dòng trong bảng → nhấn Sửa (hoặc double-click) → form Sửa hiện ra với dữ liệu cũ → user chỉnh → lưu.

Khác biệt với Thêm: form Sửa khóa mã số và loại hàng (không cho đổi), chỉ cho sửa các trường còn lại.

📖 Mô tả yêu cầu barem

Nhóm 4 — Sửa sản phẩm: User chọn dòng → mở form Sửa với dữ liệu cũ → user chỉnh → hệ thống cập nhật DSLK + SQLite + ghi Undo (lưu cả dữ liệu cũ và dữ liệu mới để Undo có thể khôi phục).

2. File _tt: sua_san_pham

from loi_thuat_toan.ket_noi_sqlite import cap_nhat_san_pham
from loi_thuat_toan.danh_sach_lien_ket import DoublyLinkedList


def sua_san_pham(danh_sach, bo_undoredo, du_lieu):
    """Cập nhật sản phẩm theo mã số."""
    try:
        # Lấy mã số sản phẩm cần sửa
        ma_so = du_lieu.get('ma_so', '').strip()
        if not ma_so:
            return False, 'Mã số không được trống.'

        # Tìm sản phẩm cần sửa trong DSLK
        sp_cu = danh_sach.tim_theo_ma_so(ma_so)
        if sp_cu is None:
            return False, f'Không tìm thấy sản phẩm mã {ma_so}.'

        # Lưu lại dữ liệu cũ (để Undo khôi phục)
        du_lieu_cu = sp_cu.chuyen_thanh_dict()

        # Tạo dict dữ liệu mới — chỉ cập nhật trường chung + trường riêng theo loại
        du_lieu_moi = {
            'ten_san_pham': du_lieu.get('ten_san_pham', '').strip(),
            'gia_co_ban':   float(du_lieu.get('gia_co_ban', 0)),
            'ton_kho':      int(du_lieu.get('ton_kho', 0)),
        }
        # Thêm trường riêng theo loại của sản phẩm
        loai_hang = sp_cu.loai_hang
        if loai_hang == 'Sách':
            du_lieu_moi['tac_gia'] = du_lieu.get('tac_gia', '').strip()
            du_lieu_moi['nha_xuat_ban'] = du_lieu.get('nha_xuat_ban', '').strip()
        elif loai_hang == 'Tạp chí':
            du_lieu_moi['so_phat_hanh'] = du_lieu.get('so_phat_hanh', '').strip()
        elif loai_hang == 'Báo giấy':
            du_lieu_moi['ngay_xuat_ban'] = du_lieu.get('ngay_xuat_ban', '').strip()
        elif loai_hang == 'Luận văn':
            du_lieu_moi['tac_gia'] = du_lieu.get('tac_gia', '').strip()
            du_lieu_moi['truong_dai_hoc'] = du_lieu.get('truong_dai_hoc', '').strip()
        elif loai_hang == 'Bản thảo':
            du_lieu_moi['tac_gia'] = du_lieu.get('tac_gia', '').strip()
            du_lieu_moi['tinh_trang'] = du_lieu.get('trang_thai', '').strip()

        # Cập nhật DSLK (RAM) + SQLite (ổ cứng)
        danh_sach.cap_nhat_node(ma_so, du_lieu_moi)
        cap_nhat_san_pham(sp_cu)

        # Ghi Undo: thao tác 'sua' + dữ liệu cũ + dữ liệu mới
        bo_undoredo['undo'].day_vao({
            'loai': 'sua',
            'du_lieu_cu': du_lieu_cu,
            'du_lieu_moi': sp_cu.chuyen_thanh_dict()
        })
        # Xóa Redo
        bo_undoredo['redo'].xoa_tat_ca()

        return True, f'Đã cập nhật "{du_lieu_moi["ten_san_pham"]}".'
    except ValueError:
        return False, 'Giá / tồn kho phải là số.'
    except Exception as loi:
        return False, f'Lỗi: {loi}'

Điểm khác so với Thêm

Điểm khác CN03 Thêm CN04 Sửa
ma_so User nhập mới Lấy từ dòng đang chọn (form khóa ô mã)
Validate trùng mã Có (vì mã mới có thể trùng mã cũ) Không cần (mã không đổi)
Tạo đối tượng mới? Có (Sach/TapChi/...) Không — chỉ cập nhật thuộc tính của đối tượng cũ
Undo entry {'loai': 'them', 'du_lieu_cu': None, 'du_lieu_moi': dict} {'loai': 'sua', 'du_lieu_cu': dict, 'du_lieu_moi': dict}
SQLite INSERT (thêm dòng mới) UPDATE (cập nhật dòng cũ)

Cú pháp mới: cap_nhat_node(ma_so, du_lieu_moi)

Hàm này của DSLK đôi (đã học ở bài 5) — duyệt tìm Node có ma_so khớp, rồi cập nhật từng trường trong du_lieu_moi dict:

def cap_nhat_node(self, ma_so, du_lieu_moi):
    node_hien_tai = self.head
    while node_hien_tai is not None:
        if hasattr(node_hien_tai.data, 'ma_so') and node_hien_tai.data.ma_so == ma_so:
            doi_tuong = node_hien_tai.data
            # Lặp từng cặp (tên_trường, giá_trị) trong dict và gán vào đối tượng
            for thuoc_tinh, gia_tri in du_lieu_moi.items():
                if hasattr(doi_tuong, thuoc_tinh):
                    setattr(doi_tuong, thuoc_tinh, gia_tri)
            return doi_tuong
        node_hien_tai = node_hien_tai.next
    return None

setattr(obj, name, value) là hàm builtin của Python — tương đương obj.name = value nhưng tên thuộc tính là chuỗi động. Dùng khi không biết trước tên thuộc tính (lấy từ dict).

3. File _dk: DieuKhienSuaSanPham

from .sua_san_pham_tt import sua_san_pham
from .sua_san_pham_ui import FormThemSuaUI


class DieuKhienSuaSanPham:
    def __init__(self, danh_sach, bo_undoredo):
        self.danh_sach = danh_sach
        self.bo_undoredo = bo_undoredo

    def mo_form_sua(self, parent, ma_so):
        """Mở form Sửa modal → chờ user chỉnh → lưu → đóng form."""
        # Tìm sản phẩm cần sửa trong DSLK
        sp_cu = self.danh_sach.tim_theo_ma_so(ma_so)
        if sp_cu is None:
            return False   # Không tìm thấy → trả về False để caller báo lỗi

        # Lấy dict dữ liệu cũ để đổ vào form
        du_lieu_cu = sp_cu.chuyen_thanh_dict()

        # Tạo form Sửa (du_lieu_cu != None → chế độ Sửa)
        form = FormThemSuaUI(parent, du_lieu_cu=du_lieu_cu)
        # Gắn sự kiện nút Lưu
        form.btn_luu.config(command=lambda: self._xu_ly_luu(form))

        # Chờ đến khi form đóng
        parent.wait_window(form.top)

        return form.ket_qua is not None

    def _xu_ly_luu(self, form):
        """Đọc dữ liệu từ form → gọi thuật toán → thông báo kết quả."""
        du_lieu = form.lay_du_lieu()
        # Đảm bảo mã số không bị đổi (form khóa ô mã nhưng phòng hờ)
        du_lieu['ma_so'] = form.du_lieu_cu['ma_so']

        ok, msg = sua_san_pham(self.danh_sach, self.bo_undoredo, du_lieu)
        if ok:
            form.ket_qua = du_lieu
            form.bao_thanh_cong(msg)
            form.dong()
        else:
            form.bao_loi(msg)

Điểm khác so với Điều khiển Thêm

  1. Nhận thêm tham số ma_so từ caller (mã của dòng đang chọn).
  2. Tìm sản phẩm trong DSLK bằng tim_theo_ma_so(ma_so).
  3. Lấy dict dữ liệu cũ → truyền vào form (kích hoạt chế độ Sửa).
  4. Trong _xu_ly_luu: đảm bảo mã số không bị đổi — gán lại du_lieu['ma_so'] = form.du_lieu_cu['ma_so'] phòng hờ.

4. Cơ chế double-click mở Sửa

Đồ án cho phép user double-click vào dòng trong bảng để mở form Sửa nhanh — tiện hơn là chọn dòng rồi nhấn nút Sửa.

Bind trong cua_so_chinh_ui.py

def bind_double_click_bang(self, ham_xu_ly):
    """Gắn sự kiện double-click vào dòng bảng → hàm xử lý (mở form Sửa)."""
    self.ui_bang.tree.bind('<Double-1>', lambda e: ham_xu_ly())

<Double-1> = double-click chuột trái (button-1). Các event chuột khác:

  • <Button-1>: click chuột trái
  • <Button-2>: click chuột giữa (wheel)
  • <Button-3>: click chuột phải
  • <Double-1>: double-click trái
  • <Triple-1>: triple-click trái
  • <ButtonRelease-1>: nhả chuột trái

Gắn bind từ cua_so_chinh_dk.py

# Trong DieuKhienCuaSoChinh.__init__
self.ui.bind_double_click_bang(self._sua_san_pham)

def _sua_san_pham(self):
    """Lấy mã đang chọn → mở form Sửa → nếu lưu → refresh_all."""
    ma = self.ui.ui_bang.lay_ma_dang_chon()
    if not ma:
        messagebox.showwarning('Chưa chọn', 'Vui lòng chọn 1 dòng để sửa.')
        return
    da_sua = self.dk_sua.mo_form_sua(self.root, ma)
    if da_sua:
        self.refresh_all()

5. Ghi lịch sử Undo khi Sửa

Khác với Thêm (chỉ có du_lieu_moi), Undo entry của Sửa chứa cả dữ liệu cũ và dữ liệu mới:

{
    'loai': 'sua',
    'du_lieu_cu': {                       # Dữ liệu TRƯỚC khi sửa
        'ma_so': 'MH001',
        'ten_san_pham': 'Truyện Kiều',
        'gia_co_ban': 50000,
        'ton_kho': 20,
        ...
    },
    'du_lieu_moi': {                      # Dữ liệu SAU khi sửa
        'ma_so': 'MH001',
        'ten_san_pham': 'Truyện Kiều (bản mới)',
        'gia_co_ban': 55000,
        'ton_kho': 25,
        ...
    }
}

Khi Undo thao tác Sửa:

  1. Lấy du_lieu_cu từ entry.
  2. Gọi cap_nhat_node(ma, du_lieu_cu) để khôi phục dữ liệu cũ.
  3. Gọi cap_nhat_san_pham(sp) để cập nhật SQLite.

Khi Redo:

  1. Lấy du_lieu_moi từ entry.
  2. Gọi cap_nhat_node(ma, du_lieu_moi) để áp dụng lại dữ liệu mới.
  3. Gọi cap_nhat_san_pham(sp).
💡 Lý do phải lưu cả dữ liệu cũ

Khác với Undo Thêm (chỉ cần mã để xóa), Undo Sửa cần toàn bộ dữ liệu cũ để khôi phục lại. Nếu chỉ lưu mã, ta không biết tên/giá/tồn kho cũ là gì để hoàn tác.

✅ Tóm tắt CN04
  • Form Sửa dùng chung UI với form Thêm (chế độ du_lieu_cu).
  • Form khóa mã số + loại hàng (không cho đổi khi sửa).
  • Thuật toán dùng cap_nhat_node của DSLK + cap_nhat_san_pham của SQLite.
  • Undo entry lưu cả dữ liệu cũ + mới để có thể khôi phục cả 2 chiều.
  • Double-click vào dòng cũng mở form Sửa (event <Double-1>).
